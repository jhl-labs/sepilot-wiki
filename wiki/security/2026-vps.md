## 2. VPS 선택 및 초기 프로비저닝

## 4. SSH 보안 강화
1. **키 쌍 생성**  
   ```bash
   ssh-keygen -t ed25519 -C "your_email@example.com"
   ```
2. **키 배포**  
   ```bash
   ssh-copy-id devuser@<server-ip>
   ```
3. **비밀번호 인증 비활성화**  
   `/etc/ssh/sshd_config` 파일을 열고 다음 옵션을 설정합니다.
   ```text
   PasswordAuthentication no
   PermitRootLogin no
   Port 2222          # 기본 22번 포트 변경 (선택)
   AllowUsers devuser
   ```
   변경 후 SSH 서비스 재시작  
   ```bash
   sudo systemctl restart sshd
   ```
4. **Fail2Ban 설치** (무차별 로그인 방어)  
   ```bash
   sudo apt install -y fail2ban
   sudo systemctl enable fail2ban
   sudo systemctl start fail2ban
   ```
위 절차는 euno.news에서 제시된 SSH 키 사용 및 `ssh-keygen`, `ssh-copy-id` 흐름을 그대로 따릅니다[[euno.news](https://euno.news/posts/ko/how-to-set-up-a-secure-vps-in-2026-beginner-friend-adb503)].

## 5. 방화벽 설정 및 네트워크 보호
### UFW(간편 방화벽) 사용 예시
```bash
# OpenSSH 포트 허용 (기본 22번 또는 위에서 변경한 포트)
sudo ufw allow OpenSSH

# HTTP/HTTPS 허용 (웹 서버 운영 시)
sudo ufw allow http
sudo ufw allow https

# 방화벽 활성화
sudo ufw enable

# 현재 규칙 확인
sudo ufw status verbose
```
UFW는 euno.news 가이드에서 권장하는 방화벽 도구이며, 위 명령어는 그대로 인용되었습니다[[euno.news](https://euno.news/posts/ko/how-to-set-up-a-secure-vps-in-2026-beginner-friend-adb503)].

### 추가 보안 옵션 (선택)
- **Rate limiting**: `sudo ufw limit OpenSSH` 로 SSH 로그인 시도 제한  
- **포트 스캐닝 방지**: `sudo ufw deny from <suspicious-ip>` 로 의심 IP 차단  

> nftables 등 다른 방화벽 솔루션에 대한 상세 설정은 **추가 조사가 필요합니다**.

## 6. 시스템 하드닝 (보안 기본 설정)
| 작업 | 설명 | 참고 |
|------|------|------|
| 불필요한 서비스 제거 | `sudo systemctl disable --now <service>` 로 사용하지 않는 데몬 비활성화 | |
| 자동 보안 업데이트 | `sudo apt install unattended-upgrades` 후 `/etc/apt/apt.conf.d/50unattended-upgrades` 설정 | |
| 파일 권한 점검 | `/etc/passwd`, `/etc/shadow` 등 민감 파일 권한 확인 (`ls -l`) | |
| 커널 파라미터 튜닝 | `/etc/sysctl.d/99-sysctl.conf` 에 `net.ipv4.ip_forward=0` 등 추가 | |
| AppArmor 활성화 | Ubuntu 기본 제공, `sudo systemctl enable apparmor && sudo systemctl start apparmor` | [AppArmor 공식 문서](https://gitlab.com/apparmor/apparmor/-/wikis/home) |

> 위 내용은 일반적인 Linux 하드닝 절차이며, 구체적인 파라미터 값은 환경에 따라 달라질 수 있어 **추가 조사가 필요합니다**.

## 7. 웹 서버 설치 및 보안
### Nginx 설치 (예시)
```bash
sudo apt install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx
```
Nginx 설치 명령은 euno.news 가이드와 동일합니다[[euno.news](https://euno.news/posts/ko/how-to-set-up-a-secure-vps-in-2026-beginner-friend-adb503)].

### 보안 헤더 적용 (간단 예시)
`/etc/nginx/conf.d/security.conf` 파일에 다음을 추가:
```nginx
add_header X-Content-Type-Options nosniff;
add_header X-Frame-Options SAMEORIGIN;
add_header Content-Security-Policy "default-src 'self'";
add_header Referrer-Policy no-referrer;
```
### HTTPS 적용 – Let’s Encrypt
```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d example.com -d www.example.com
```
*Certbot 사용법은 공식 문서([certbot.eff.org](https://certbot.eff.org/))를 참고하세요.*

### 로그 관리
- 접근 로그: `/var/log/nginx/access.log`  
- 오류 로그: `/var/log/nginx/error.log`  
`journalctl -u nginx` 로 시스템 로그와 연동 가능

## 9. 백업 및 복구 전략
1. **스냅샷 기반 백업** – VPS 제공업체 콘솔에서 정기 스냅샷 생성 (빠른 복구).  
2. **파일 기반 백업** – `rsync`와 `cron`을 이용한 일일 백업 스크립트 예시:
   ```bash
   # /usr/local/bin/backup.sh
   rsync -a --delete /var/www/ /backup/www/
   rsync -a --delete /etc/ /backup/etc/
   ```
   ```bash
   # crontab -e
   0 2 * * * /usr/local/bin/backup.sh >> /var/log/backup.log 2>&1
   ```
3. **백업 암호화** – `gpg` 로 백업 파일 암호화 후 외부 저장소(S3 등) 전송.  
4. **복구 테스트** – 정기적으로 스냅샷 또는 rsync 백업을 새로운 인스턴스에 복원해 보는 절차를 수행.

> 외부 스토리지 연동 방법(AWS S3, Wasabi 등)은 제공업체 API에 따라 다르므로 **추가 조사가 필요합니다**.

## 10. 컨테이너 및 DevOps 워크플로우 준비
### Docker 엔진 설치
```bash
sudo apt install -y ca-certificates curl gnupg
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io
sudo systemctl enable docker && sudo systemctl start docker
```
공식 Docker 설치 가이드는 [Docker Docs](https://docs.docker.com/engine/install/ubuntu/)를 참고하세요.

### Docker Compose
```bash
sudo curl -L "https://github.com/docker/compose/releases/download/v2.27.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
docker-compose --version
```
