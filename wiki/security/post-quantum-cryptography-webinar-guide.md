## 1. 서론
- **문서 목적**: 양자 컴퓨팅 시대에 대비하기 위해 2023 년 “Future‑Ready Security” 웨비나에서 다룬 핵심 내용을 정리하고, 조직이 실무에 바로 적용할 수 있는 로드맵을 제공한다.  
- **대상 독자**: 보안 담당자, IT 운영팀, 암호화 설계자, 경영진 등 양자 위협에 관심이 있는 모든 기술 리더.  
- **포스트‑양자 암호학(PQC) 도입 배경**: 양자 컴퓨터가 RSA·ECC 등 현재 주류 암호를 무력화할 가능성이 높아짐에 따라, “harvest‑now‑decrypt‑later” 공격에 대비한 사전 방어가 필요하다.  
- **웨비나와 본 가이드 연계성**: 웨비나에서 제시된 **하이브리드 암호화** 전략을 실무에 적용하기 위한 단계별 체크리스트와 조직 현황 파악 방법을 정리한다.  

## 2. 양자 위협 개요
| 위협 요소 | 설명 |
|---|---|
| **Harvest‑Now‑Decrypt‑Later** | 공격자는 현재 암호화된 데이터를 수집·저장하고, 양자 컴퓨터가 실용화되면 나중에 복호화한다. 이는 오늘 전송되는 민감 데이터가 수년 뒤에 노출될 위험을 의미한다.[[euno.news](https://euno.news/posts/ko/preparing-for-the-quantum-era-post-quantum-cryptog-d89b96)] |
| **양자 컴퓨터가 RSA/ECC를 무력화** | Shor 알고리즘 등 양자 알고리즘은 현재 널리 사용되는 공개키 암호(RSA, ECC)를 다항 시간에 풀 수 있다. 따라서 “영원히 안전”하다고 가정할 수 없다.[[euno.news](https://euno.news/posts/ko/preparing-for-the-quantum-era-post-quantum-cryptog-d89b96)] |
| **민감 데이터 보존 기간** | 금융 기록, 지적 재산, 정부 통신 등 수년·수십 년 동안 기밀을 유지해야 하는 데이터는 양자 위협에 즉시 대비해야 한다.[[euno.news](https://euno.news/posts/ko/preparing-for-the-quantum-era-post-quantum-cryptog-d89b96)] |

## 3. 포스트‑양자 암호학 기본
- **PQC 정의**: 양자 컴퓨터에 저항하도록 설계된 암호 알고리즘 집합.  
- **표준화 진행 상황**: 현재 **NIST**가 PQC 표준화를 진행 중이다. (구체적인 진행 단계는 추가 조사가 필요합니다.)  
- **대표적인 양자‑저항 알고리즘**  
  - **ML‑KEM**: 웨비나에서 하이브리드 암호화에 활용되는 주요 KEM(Key Encapsulation Mechanism) 중 하나다.[[euno.news](https://euno.news/posts/ko/preparing-for-the-quantum-era-post-quantum-cryptog-d89b96)]  
  - **Kyber, Dilithium 등**: NIST 후보 알고리즘이지만, 본 문서에서는 구체적인 내용이 제공되지 않아 추가 조사가 필요합니다.  

## 4. 웨비나 핵심 내용 요약
| 항목 | 내용 |
|---|---|
| **발표자·세션 구성** | 보안·네트워킹 리더를 대상으로 한 실시간 세션. Zscaler 등 솔루션 제공 업체가 구현 사례를 공유했다.[[euno.news](https://euno.news/posts/ko/preparing-for-the-quantum-era-post-quantum-cryptog-d89b96)] |
| **양자 위협 인식·평가** | “Harvest‑Now‑Decrypt‑Later” 공격 위험이 급증하고 있음을 강조. |
| **하이브리드 암호화 전략** | 전통 암호화와 **ML‑KEM** 같은 양자‑저항 알고리즘을 결합해 기존 시스템을 방해하지 않으면서 보안을 강화한다. |
| **실무 적용 로드맵** | 1) 민감 데이터 식별, 2) 암호화 적용 지점 파악, 3) 하이브리드 파일럿 실행, 4) 전사적 PQC 전환 계획 수립을 제시.[[euno.news](https://euno.news/posts/ko/preparing-for-the-quantum-era-post-quantum-cryptog-d89b96)] |

## 5. 하이브리드 암호화 전략
1. **전통 암호화 + PQC 결합 방식**  
   - 기존 TLS/HTTPS 등에서 RSA/ECC 기반 키 교환을 유지하면서, 세션 키를 **ML‑KEM** 로도 캡슐화한다.  
2. **ML‑KEM 기반 구현 원리**  
   - 클라이언트는 서버의 공개 KEM 키로 세션 키를 캡슐화하고, 서버는 비밀 KEM 키로 복호화한다. 기존 RSA/ECC와 병행해 두 개의 키 교환이 동시에 수행된다.  
3. **성능·호환성 고려사항**  
   - 현재 대부분의 하드웨어·소프트웨어 스택은 ML‑KEM을 네이티브로 지원하지 않으므로, 라이브러리 레이어에서 래핑하거나 프록시 형태로 적용한다.  
   - 하이브리드 방식은 기존 프로토콜을 그대로 사용하므로 서비스 중단 없이 점진적 전환이 가능하다.  

## 6. 실무 적용 로드맵
### 단계 1️⃣: 민감 데이터 및 보호 기간 식별
- 데이터 카탈로그 작성 → 보관 기간(예: 5 년, 10 년) 정의.  
- “양자 위협에 노출될 가능성”이 높은 데이터군을 우선순위에 올린다.  

### 단계 2️⃣: 시스템 전반 암호화 사용 지점 파악
- **스토리지**(디스크 암호화, DB 암호화)  
- **전송**(TLS, VPN, API)  
- **인증**(PKI, JWT 서명)  
- 각 지점에서 현재 사용 중인 알고리즘·버전을 기록한다.  

### 단계 3️⃣: 하이브리드 암호화 파일럿 프로젝트 실행
- 파일럿 대상 서비스 선정(예: 내부 API 게이트웨이).  
- ML‑KEM 라이브러리(예: Open Quantum Safe)와 기존 TLS 스택을 연동.  
- 성능·호환성 테스트 후, 보안 로그와 키 관리 절차를 검증한다.  

### 단계 4️⃣: 전사적 PQC 전환 계획 수립
- 파일럿 결과를 바탕으로 전사적 적용 범위와 일정 정의.  
- 키 관리(KMS)·키 교체 정책을 업데이트하고, 감사·컴플라이언스 체크포인트를 마련한다.  

## 7. 조직 내 현황 파악 방법
| 체크리스트 항목 | 설명 |
|---|---|
| **데이터 자산 조사** | 데이터베이스, 파일 스토리지, 백업 등 모든 데이터 저장소를 목록화한다. |
| **암호화 적용 포인트 매핑** | 스토리지·전송·인증 각각에 사용되는 암호화 프로토콜·알고리즘을 매핑한다. |
| **현재 알고리즘·버전 현황** | RSA‑2048, ECC‑P‑256 등 현재 운영 중인 키 길이·버전을 기록한다. |
| **규제·컴플라이언스 요구사항** | 금융·정부 규제에서 요구하는 최소 암호화 수준을 확인한다. |

## 8. 정책·규정·표준 연계
- **NIST PQC 로드맵**: NIST가 발표한 PQC 표준 초안 및 최종 표준 채택 일정은 조직 정책에 반영해야 한다(구체적인 일정은 추가 조사가 필요합니다).  
- **국내·외 규제**: 금융권·정부기관 등에서 양자‑저항 암호 적용을 권고하거나 의무화하는 움직임이 있다. 최신 규제 동향은 정기적으로 검토한다.  
- **보안 정책에 PQC 요소 통합**  
  - 암호화 알고리즘 선택 가이드에 “양자‑저항 알고리즘 우선 적용” 조항 추가.  
  - 키 교체 주기를 “양자 위험 평가 결과에 따라” 조정한다.  
- **감사·컴플라이언스 체크포인트**  
  - 알고리즘·키 길이 검증, 하이브리드 구현 로그, KMS 접근 제어 등을 점검 항목에 포함한다.

## 10. 전환 체크리스트 & 베스트 프랙티스
- **기술 측면**  
  - 하이브리드 구현 테스트 → 성능 임계치 확인  
  - 키 관리 시스템(KMS)에서 PQC 키 유형 지원 여부 검증  
- **운영 측면**  
  - 파일럿 단계에서 운영팀·보안팀·개발팀 간 협업 프로세스 정의  
  - 장애 발생 시 기존 RSA/ECC 로 롤백 가능한 절차 마련  
- **인력 측면**  
  - PQC 개념·알고리즘 교육 프로그램 운영  
  - 벤더·오픈소스 커뮤니티와 정기적인 기술 교류 유지  
- **보안 베스트 프랙티스**  
  - 키 교체 주기를 최소 2 년 이하로 설정(양자 위험 평가 기반)  
  - 모든 신규 서비스는 기본 하이브리드 암호화 적용을 원칙으로 함  

## 11. 향후 전망 및 지속적 업데이트
- **양자 컴퓨팅 로드맵**: 양자 하드웨어가 실용화되기까지는 “수년” 수준으로 예측되며, 정확한 시점은 지속적인 연구·시장 동향 파악이 필요하다(추가 조사가 필요합니다).  
- **PQC 표준·알고리즘 업데이트 추적**: NIST 발표, 국제 표준 기구(IETF, ISO) 업데이트를 구독하고, 조직 내부 보안 로드맵에 반영한다.  
- **양자 대비 보안 문화 구축**  
  - 정기적인 “양자 위협 인식” 워크숍 개최  
  - KPI에 “양자‑저항 암호 적용 비율”을 포함  

## 12. 부록
### 용어 정의
- **PQC (Post‑Quantum Cryptography)**: 양자 컴퓨터에 저항하도록 설계된 암호 알고리즘 집합.  
- **KEM (Key Encapsulation Mechanism)**: 공개키 기반으로 세션 키를 안전하게 전달하는 메커니즘.  
- **Hybrid Encryption**: 전통 암호화와 PQC 알고리즘을 동시에 사용해 보안을 강화하는 방식.  
- **Harvest‑Now‑Decrypt‑Later**: 현재 암호화된 데이터를 저장해 두고, 미래에 양자 컴퓨터로 복호화하는 공격 전략.

### FAQ
1. **Q: 기존 시스템에 바로 하이브리드 암호화를 적용할 수 있나요?**  
   **A:** 네, 하이브리드 방식은 기존 TLS/HTTPS 스택을 그대로 유지하면서 추가적인 KEM 레이어만 삽입하므로 서비스 중단 없이 단계적으로 적용할 수 있다.  

2. **Q: 모든 서비스에 즉시 PQC를 적용해야 하나요?**  
   **A:** 데이터 보존 기간이 길고 민감도가 높은 서비스부터 우선 적용하고, 파일럿을 통해 성능·호환성을 검증한 뒤 전사적으로 확대한다.  

3. **Q: 현재 NIST 표준이 확정되면 기존 키는 어떻게 교체해야 하나요?**  
   **A:** 키 교체 정책에 “표준 확정 시 2 년 이내 교체” 조항을 두고, KMS에서 자동 교체 스케줄링을 설정한다. (구체적 일정은 NIST 발표에 따라 업데이트 필요)
