## 개요
이 문서는 2026년 기준 최신 Ubuntu LTS 배포판을 기준으로, **VPS(Virtual Private Server)를 처음 배포하고 보안을 강화하고자 하는 개발자·시스템 관리자·DevOps 초보자**를 위한 단계별 가이드를 제공합니다.

## 사전 준비
- **필요 계정·권한**: 루트(root) 혹은 `sudo` 권한이 있는 일반 사용자 계정이 필요합니다.  
- **지원 배포판**: Ubuntu 22.04 LTS(또는 최신 LTS) 권장 [Ubuntu 공식 문서](https://ubuntu.com/server/docs).  
- **네트워크 정보**: 서버 공인 IP, DNS 설정(예: Cloudflare) 등 기본 네트워크 정보를 사전에 확보합니다.  
- **SSH 키 도구**: 로컬 머신에 `ssh-keygen`이 포함된 OpenSSH 클라이언트가 설치되어 있어야 합니다.  

---

## 초기 서버 설정
1. **원격 접속**  
   ```bash
   ssh user@server-ip
   ```  
2. **시스템 업데이트·업그레이드**  
   ```bash
   sudo apt update && sudo apt upgrade
   ```  
3. **기본 유틸리티 설치** (선택 사항)  
   `curl`, `wget`, `git` 등 개발·운영에 자주 쓰이는 패키지를 미리 설치합니다.  

---

## SSH 보안 강화
- **키 기반 인증**  
  ```bash
  ssh-keygen          # 로컬에서 키 쌍 생성
  ssh-copy-id user@server-ip   # 공개키 전송
  ```  
- **비밀번호 로그인 비활성화**  
  `/etc/ssh/sshd_config` 파일에서 `PasswordAuthentication no` 로 설정하고 SSH 서비스를 재시작합니다.  
- **포트 변경·접속 제한**  
  기본 포트 22 대신 다른 포트(예: 2222)로 변경하고, `AllowUsers` 지시어로 허용 사용자만 지정합니다.  
- **Fail2Ban 설치·기본 설정** (추가 조사가 필요합니다) – 반복적인 로그인 시도를 차단하는 도구로, 기본 `jail.conf`에 `sshd` 항목을 활성화합니다.  

---

## 방화벽 구성 (UFW)
1. **UFW 설치·활성화**  
   ```bash
   sudo apt install ufw
   sudo ufw allow OpenSSH   # SSH 포트 허용
   sudo ufw enable
   sudo ufw status
   ```  
2. **필수 포트 추가**  
   - HTTP: `sudo ufw allow 80/tcp`  
   - HTTPS: `sudo ufw allow 443/tcp`  
3. **로그 관리**  
   UFW 로그는 `/var/log/ufw.log`에 기록되며, `sudo ufw logging on` 으로 상세 로그를 활성화할 수 있습니다.

## 시스템 하드닝
- **불필요 서비스 비활성화**  
  `systemctl list-unit-files --type=service` 로 실행 중인 서비스를 확인하고, 사용하지 않는 서비스는 `disable` 및 `stop` 처리합니다.  
- **커널 보안 파라미터** (`/etc/sysctl.conf`)  
  - IP 스푸핑 방지: `net.ipv4.conf.all.rp_filter=1`  
  - ICMP 리다이렉트 차단: `net.ipv4.conf.all.accept_redirects=0`  
  적용 후 `sudo sysctl -p` 로 재로드합니다.  
- **자동 보안 업데이트**  
  `unattended-upgrades` 패키지를 설치하고, `/etc/apt/apt.conf.d/50unattended-upgrades` 에 보안 업데이트만 적용하도록 설정합니다.  
- **파일 권한·AppArmor**  
  기본 파일 권한을 최소 권한 원칙에 따라 설정하고, Ubuntu 기본 AppArmor 프로파일을 활성화합니다 [AppArmor 공식 문서](https://gitlab.com/apparmor/apparmor/-/wikis/home).

## 웹 서버 설치 및 보안
### Nginx vs Apache 선택
- **Nginx**: 높은 동시 연결 처리와 정적 파일 서빙에 강점.  
- **Apache**: 모듈 기반 확장성이 필요할 때 유리.  

### 설치 예시 (Nginx)  
```bash
sudo apt install nginx
sudo systemctl enable nginx
sudo systemctl start nginx
```  

### SSL/TLS 적용
- **Let’s Encrypt** 자동 인증서 발급 (`certbot`) 사용을 권장합니다 [Certbot 공식 가이드](https://certbot.eff.org/).  
- 인증서 갱신은 `certbot renew --dry-run` 로 자동화합니다.  

### 보안 헤더·기본 모듈
- `add_header X-Content-Type-Options nosniff;`  
- `add_header X-Frame-Options SAMEORIGIN;`  
- `add_header Content-Security-Policy "default-src 'self'";`

## 모니터링 및 로깅
- **로그 위치**: `/var/log/` 아래에 주요 서비스 로그가 저장됩니다.  
- **fail2ban 연동**: `fail2ban` 은 자체 로그와 `/var/log/auth.log` 를 분석해 차단 규칙을 자동 생성합니다.  
- **간단 모니터링**: `uptime`, `free -h` 등 기본 명령어로 시스템 상태를 확인할 수 있습니다.  
- **Prometheus Node Exporter** (추가 조사가 필요합니다) – 메트릭 수집을 위한 경량 에이전트이며, Grafana와 연동해 시각화할 수 있습니다.

## DevOps 워크플로우 준비
- **Docker 설치**  
  ```bash
  sudo apt install docker.io
  sudo systemctl enable docker
  sudo systemctl start docker
  ```  
- **컨테이너 실행 예시**  
  `docker run -d -p 80:80 nginx` 로 Nginx 컨테이너를 빠르게 배포합니다.  
- **CI/CD 파이프라인**  
  - GitHub Actions, GitLab CI 등 클라우드 기반 워크플로우를 활용해 자동 빌드·배포를 구현합니다.  
- **인프라 테스트 도구**  
  - **Terraform**: 인프라 코드화(IaC) [Terraform 공식 문서](https://developer.hashicorp.com/terraform).  
  - **Ansible**: 설정 자동화·배포 [Ansible 공식 문서](https://docs.ansible.com/).

## 보안 점검 체크리스트
1. **SSH**  
   - 키 기반 인증 적용 여부  
   - 비밀번호 로그인 비활성화  
   - 포트 변경·접속 제한 설정  
2. **방화벽**  
   - UFW 정책 적용 및 로그 확인  
3. **시스템 하드닝**  
   - 불필요 서비스 제거  
   - `sysctl` 보안 파라미터 적용  
   - 자동 보안 업데이트 활성화  
4. **웹 서버**  
   - 최신 패키지 사용 여부  
   - SSL 인증서 정상 발급·갱신  
   - 보안 헤더 적용 여부  
5. **모니터링·로깅**  
   - 주요 로그 파일 권한 검토  
   - fail2ban 연동 확인  
6. **Docker**  
   - 최신 이미지 사용 및 최소 권한 실행  

*자동화 스크립트* (예: Bash 또는 Ansible 플레이북) 를 활용하면 위 체크리스트를 반복적으로 실행할 수 있습니다.
