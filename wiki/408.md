## 1. 개요
- **취약점 식별**: GHSA‑QR2G‑P6Q7‑W82M (Coinbase x402 SDK Solana SVM 결제 검증)  
- **공개일**: 2026‑03‑07  
- **CVSS 점수**: 9.9 (Critical) – 매우 높은 위험도 [🔗EUNO.NEWS](https://euno.news/posts/ko/ghsa-qr2g-p6q7-w82m-ghsa-qr2g-p6q7-w82m-critical-p-e0c755)  
- **주요 영향**: Solana (SVM) 기반 자동 HTTP 402 결제 검증 로직이 서명 검증 오류(CWE‑347)로 인해 결제 요구 사항을 우회할 수 있음.

## 2. 기술적 세부 정보
| 항목 | 내용 |
|------|------|

| **핵심 원인** | Ed25519 서명 검증 구현 오류 – 서명이 올바르지 않아도 검증을 통과함 (CWE‑347) |
| **공격 벡터** | 네트워크 기반 위조 서명 요청 → 결제 서명 위조 후 API·리소스·디지털 상품 무단 접근 |
| **피해 가능 시나리오** | 블록체인에 거래를 정산하지 않은 상태에서 유료 API, 컴퓨팅 리소스, 디지털 상품을 무료로 이용 |

## 3. 영향을 받는 제품 및 버전
- **Node.js** – `@x402/svm` (npm) **< 2.6.0**  
- **Python** – `x402` (PyPI) **< 2.3.0**  
- **Go** – `github.com/coinbase/x402/go` **< 2.5.0**  

위 버전을 사용하는 모든 **facilitator** 서비스가 직접적인 영향을 받음 [🔗EUNO.NEWS].

## 4. 위험 평가 및 비즈니스 영향
1. **재정 손실** – 결제 검증 회피를 통해 비용이 청구되지 않아 서비스 제공자가 손실을 입음.  
2. **서비스 남용** – 무단 API 호출·컴퓨팅 리소스 사용으로 서비스 가용성 저하 및 과부하 위험.  
3. **신뢰도 하락** – 블록체인 기반 결제 시스템의 무결성에 대한 신뢰 손상.  
4. **법적·규제 위험** – 결제 회피가 금융 규제 위반으로 이어질 가능성.

## 5. 탐지 및 모니터링 지침
- **서명 로그 분석**  
  - 서명 검증 요청에 포함된 `signature` 필드 값이 비정상적으로 반복되거나 변조된 경우 경고.  
- **IP·거래량 감시**  
  - 단일 IP에서 비정상적으로 높은 체인 거래량(특히 402 응답 후 결제 없이) 감지.  
- **SIEM 연동**  
  - 위 이벤트를 **Syslog**, **Elastic**, **Splunk** 등과 연동하여 실시간 알림 설정.  
- **메트릭**  
  - `facilitator.payment_verification_failure` (실패 카운트)와 `facilitator.unsettled_requests` (정산되지 않은 요청) 지표를 대시보드에 추가.

## 6. 완화·대응 전략
1. **패치 적용** – 아래 **업그레이드 절차**에 따라 최신 버전(≥ 2.6.0 / 2.3.0 / 2.5.0)으로 즉시 업데이트 [🔗EUNO.NEWS].  
2. **임시 방어**  
   - Solana (SVM) 결제 기능을 **비활성화**하고, 필요 시 **EVM‑전용 결제**로 전환.  
   - 결제 검증 로직 앞에 **추가 서명 검증**(예: 서버‑사이드 Ed25519 검증 라이브러리) 적용.  
3. **접근 제한**  
   - 의심 IP 차단, **레이트 리밋**(예: 1분당 10건) 적용.  
   - **WAF** 규칙 추가: `POST /payment/verify` 에서 비정상적인 `signature` 길이/형식 차단.  

## 7. 업그레이드 절차
### Node.js / TypeScript
- `npm install @x402/svm@latest`  # 버전 2.6.0 이상 설치  
- `package-lock.json` 혹은 `yarn.lock` 업데이트 후 서비스 재시작  

### Python
- `pip install x402 --upgrade`  # 버전 2.3.0 이상 설치  
- 가상환경(`venv`/`conda`) 재활성화 후 애플리케이션 재시작  

### Go
- `go get github.com/coinbase/x402/go@v2.5.0`  # `go.mod`에 패치 버전 명시  
- `go mod tidy` 실행 후 바이너리 재빌드 및 재배포  

> **주의**: 모든 **facilitator** 서비스가 새 버전을 로드하도록 **재시작**이 필요합니다 [🔗EUNO.NEWS].

## 8. 검증 및 테스트
1. **서명 검증 정상 동작 확인**  
   - 정상 서명과 변조 서명을 각각 전송해 `HTTP 402` 응답이 올바르게 반환되는지 확인.  
2. **회귀 테스트**  
   - 기존 결제 흐름(정상 결제, 실패 결제) 시나리오를 자동화 테스트 스위트에 포함.  
3. **로드 테스트**  
   - 정상 트래픽 수준에서 `facilitator`가 과부하 없이 서명 검증을 수행하는지 부하 테스트 수행.

## 10. 부록
### CWE‑347 – Improper Verification of Cryptographic Signature
- 서명 검증 로직이 입력값을 충분히 검증하지 않아 위조된 서명이 통과될 수 있음.  
- 권장 방어: 검증 전 **입력 길이·형식** 검증, **표준 라이브러리** 사용, **시계열(Nonce)** 포함.

### CVSS 계산 근거
- **Base Score** 9.9 (Critical) – 공격 복잡도 낮음, 권한 상승 없음, 기밀성·무결성·가용성 모두 심각.
