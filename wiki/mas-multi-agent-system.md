# MAS (Multi Agent System)

## 개요
Multi Agent System(MAS)은 **여러 개의 자율적인 에이전트가 상호작용**하면서 복잡한 문제를 해결하거나 목표를 달성하도록 설계된 시스템입니다. 각 에이전트는 자체적인 지식, 목표, 행동 능력을 가지고 있으며, 협업·경쟁·협상 등을 통해 전체 시스템의 행동을 형성합니다.

## 핵심 개념
- **에이전트(Agent)**: 독립적인 의사결정 주체. 센서, 액추에이터, 내부 상태, 행동 정책을 포함합니다.
- **환경(Environment)**: 에이전트가 인식하고 행동하는 외부 세계.
- **상호작용(Interaction)**: 메시지 교환, 관찰, 행동 등 에이전트 간의 커뮤니케이션 방식.
- **협업(Cooperation) vs 경쟁(Competition)**: 목표 달성을 위해 협력하거나 자원을 놓고 경쟁할 수 있습니다.
- **학습(Learning)**: 강화학습, 진화 알고리즘 등으로 에이전트가 경험을 통해 정책을 개선합니다.
- **조정(Coordination) 메커니즘**: 계약, 경매, 토큰 기반 등 에이전트 간 행동을 조율하는 방법.

## 장점
- **확장성**: 새로운 에이전트를 추가해도 시스템 전체를 재설계할 필요가 없습니다.
- **유연성**: 각 에이전트가 독립적으로 업데이트·교체될 수 있어 변화에 빠르게 대응합니다.
- **복원력**: 일부 에이전트가 실패해도 전체 시스템이 계속 동작할 수 있습니다.
- **분산 처리**: 작업을 여러 에이전트에 분산시켜 효율성을 높입니다.

## 도전 과제
- **통신 오버헤드**: 에이전트 간 빈번한 메시징은 네트워크 부하를 초래할 수 있습니다.
- **협업 설계**: 목표 충돌을 방지하고 효율적인 협업 메커니즘을 설계하는 것이 어렵습니다.
- **보안·프라이버시**: 분산된 에이전트가 민감한 데이터를 다룰 경우 보안 위험이 증가합니다.
- **디버깅·테스팅**: 다수의 자율적인 구성 요소가 상호작용하므로 시스템 전체를 테스트하고 디버깅하기가 복잡합니다.

## 활용 사례
| 분야 | 사례 | 설명 |
|------|------|------|
| **로보틱스** | 다중 로봇 협업 | 물류창고에서 여러 로봇이 협력해 물품을 이동·정렬합니다. |
| **스마트 시티** | 교통 신호 제어 | 교통 신호 에이전트가 실시간 교통 흐름을 분석해 신호 타이밍을 조정합니다. |
| **게임 AI** | NPC 팀 행동 | 게임 내 적군이 팀을 이루어 전술을 수행합니다. |
| **금융** | 포트폴리오 관리 | 여러 투자 에이전트가 서로 다른 전략으로 포트폴리오를 최적화합니다. |
| **헬스케어** | 환자 모니터링 | 다양한 센서 에이전트가 환자 데이터를 수집·분석해 맞춤형 치료를 제안합니다. |
