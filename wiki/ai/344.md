## 1. 소개
### 1.1 프로젝트 배경 및 목표
AI‑드리븐 알고리즘 탐색은 기존 수작업 설계보다 빠르게 새로운 솔루션을 찾아낼 수 있는 가능성을 제시합니다. **SkyDiscover**는 이러한 탐색 과정을 **모듈형**으로 정형화하여, 연구자와 엔지니어가 손쉽게 실험을 반복하고, 결과를 재현하며, 클라우드 비용을 최소화하도록 설계되었습니다 ([euno.news](https://euno.news/posts/ko/skydiscover-an-open-framework-for-llm-driven-algor-f20ef7)).

### 1.2 LLM‑driven 알고리즘 디스커버리의 필요성
- 복잡한 수학·시스템·프로그래밍·멀티모달 문제에 대한 **창의적 아이디어**를 LLM이 빠르게 제시  
- 자동화된 **평가·선택 루프**를 통해 인간 개입을 최소화  
- 대규모 클라우드 환경에서 **전송 비용·KV‑cache 압력**을 절감 (전송 비용 41 %↓, KV‑cache 압력 29 %↓) ([euno.news](https://euno.news/posts/ko/skydiscover-an-open-framework-for-llm-driven-algor-f20ef7))

### 1.3 대상 독자와 활용 범위
- **연구자**: 새로운 알고리즘 탐색 파이프라인을 설계·검증하고 논문에 활용  
- **엔지니어**: 프로덕션 환경에서 자동화된 최적화 루프를 구축  
- **학생·교육자**: LLM 기반 탐색 흐름을 학습·실습

## 2. 프레임워크 개요
### 2.1 전체 디스커버리 루프와 4 단계 구조
SkyDiscover는 **디스커버리 루프**를 다음 네 단계로 분해합니다.

| 단계 | 역할 |
|------|------|
| **Context Builder** | 문제 정의·컨텍스트 생성, LLM에 전달할 프롬프트를 구성 |
| **Solution Generator** | LLM을 호출해 후보 알고리즘·솔루션을 생성 |
| **Evaluator** | 자동화된 테스트·벤치마크를 통해 후보 품질을 평가 |
| **Selector** | 평가 결과를 바탕으로 다음 라운드에 사용할 후보를 선택 |

각 단계는 **교체 가능**하도록 인터페이스가 정의돼 있어, 사용자는 자신만의 구현을 플러그인 형태로 삽입할 수 있습니다.

### 2.2 모듈형 설계 철학 및 교체 가능성
- **확장성**: 새로운 알고리즘 탐색 전략(AdaEvolve, EvoX 등)을 손쉽게 추가  
- **재현성**: 파이프라인 전체를 선언형 설정 파일(`yaml`/`json`)로 관리  
- **클라우드 효율성**: 비용·캐시 사용량을 모니터링하고 자동으로 최적화  

### 2.3 주요 설계 원칙
- **플러그인 기반**: 각 구성 요소는 `Base*` 클래스를 상속해 구현  
- **표준화된 메트릭**: 평가 단계에서 수집되는 메트릭은 공통 스키마를 따름  
- **CI/CD 친화**: GitHub Actions·GitLab CI와 연동 가능한 CLI 제공

## 3. 핵심 구성 요소
### 3.1 Context Builder
- **문제 정의**: 도메인‑특화 스키마(수학, 시스템, 프로그래밍 등)와 입력 데이터를 받아 프롬프트 템플릿에 삽입  
- **프롬프트 템플릿**: Jinja2‑style 템플릿을 사용해 변수(`{{problem}}`, `{{constraints}}`)를 동적으로 채움  
- **데이터 전처리**: 텍스트 정규화·샘플링을 통해 LLM 입력 길이를 최적화  

### 3.2 Solution Generator
- **LLM 호출 인터페이스**: OpenAI, Anthropic, Azure 등 다양한 LLM API를 추상화  
- **핵심 파라미터**: `temperature`, `max_tokens`, `top_p` 등은 설정 파일에서 조정 가능  
- **후보 생성 전략**: 단일 샘플·다중 샘플·샘플링 기반 탐색을 지원  

### 3.3 Evaluator
- **성능/품질 지표**: 실행 시간, 메모리 사용량, 정확도·정밀도·리콜 등  
- **자동화된 테스트**: Docker·Podman 컨테이너 안에서 격리 실행, 결과를 JSON 형태로 수집  
- **메트릭 수집**: Prometheus exporter와 연동해 실시간 대시보드 제공  

### 3.4 Selector
- **후보 선택 기준**: 다중 목표 최적화(예: 성능 + 비용) 혹은 단일 목표(예: 최고 정확도)  
- **라운드‑트립 로직**: 평가 결과를 피드백으로 Context Builder에 전달, 다음 라운드 프롬프트를 재구성  
- **다중 후보 유지**: Top‑K 전략을 통해 여러 후보를 동시에 진행 가능

## 4. 구현 상세
### 4.1 레포지토리 구조 (예시)
```
skydiscover/
├─ skydiscover/          # 핵심 파이썬 패키지
│   ├─ context/
│   ├─ generator/
│   ├─ evaluator/
│   └─ selector/
├─ examples/            # 사용 예시 스크립트
├─ tests/               # 자동화 테스트
├─ Dockerfile           # 컨테이너 이미지 정의
├─ pyproject.toml       # 의존성 관리
└─ README.md
```

### 4.2 의존성 관리
- **Python**: `>=3.9` (pydantic, jinja2, httpx, docker)  
- **LLM API**: OpenAI SDK, Anthropic SDK 등 (선택적)  
- **Docker**: 실행 격리를 위한 기본 이미지 (`python:3.11-slim`)  

### 4.3 실행 파이프라인
1. **CLI**: `skydiscover run --config config.yaml`  
2. **스크립트**: `python -m skydiscover.pipeline config.yaml`  
3. **CI/CD**: GitHub Actions 워크플로 (`skydiscover.yml`)에서 `skydiscover run` 명령을 호출해 자동 테스트·벤치마크 수행

## 5. 내장 알고리즘
### 5.1 AdaEvolve
- **핵심 아이디어**: 탐색 단계에서 **적응형 탐색 비율**을 동적으로 조정해 탐색 공간을 효율적으로 축소  
- **파라미터**: `exploration_rate`, `decay_factor` 등은 설정 파일에서 지정  
- **적용 사례**: 수학 문제(예: Frontier‑CS 172개)에서 평균 **+34 %** 성능 개선을 달성 ([euno.news](https://euno.news/posts/ko/skydiscover-an-open-framework-for-llm-driven-algor-f20ef7))

### 5.2 EvoX
- **Self‑modifying 탐색**: 후보 알고리즘이 자체 코드를 수정·재생성하여 **다중 단계 진화**를 수행  
- **장점**: 복잡한 제약 조건을 가진 시스템 작업에서 **AlphaEvolve**와 동등하거나 우수한 성능을 보임  
- **제한점**: 자체 수정 로직이 복잡해 디버깅 비용이 증가할 수 있음  

> **리소스**: AdaEvolve·EvoX 논문 URL은 현재 제공되지 않아 **추가 조사가 필요합니다**.

## 7. 사용 가이드
### 7.1 설치 및 초기 설정
```bash
# 1. 레포지토리 클론
git clone https://github.com/skydiscover/skydiscover.git
cd skydiscover

# 2. 의존성 설치 (pipenv/poetry 등)
pip install -e .

# 3. 기본 설정 파일 복사
cp examples/config.yaml ./config.yaml
```
> 실제 GitHub URL은 현재 공개되지 않아 **추가 조사가 필요합니다**.

### 7.2 기본 워크플로우 예시
```yaml
# config.yaml (간략 예시)
context:
  problem: "최소 비용 경로 탐색"
  constraints:
    - "시간 제한 2초"
    - "메모리 제한 256MB"

generator:
  model: "gpt-4o"
  temperature: 0.7
  max_tokens: 1024

evaluator:
  benchmarks:
    - "runtime"
    - "memory"
    - "accuracy"

selector:
  strategy: "top_k"
  k: 5
```
```bash
# 파이프라인 실행
skydiscover run --config config.yaml
```
위 명령은 **Context → Generator → Evaluator → Selector** 순으로 자동 진행합니다.

### 7.3 커스텀 모듈 구현
1. `skydiscover/context/base.py`에 `BaseContextBuilder` 상속  
2. `skydiscover/generator/base.py`에 `BaseSolutionGenerator` 상속  
3. `setup.cfg` 혹은 `pyproject.toml`에 entry‑point 등록 (`skydiscover.plugins`)  

```python
# 예시: CustomContextBuilder
from skydiscover.context.base import BaseContextBuilder

class MyContextBuilder(BaseContextBuilder):
    def build(self, problem_spec):
        # 사용자 정의 로직
        return {"prompt": f"Solve: {problem_spec['description']}"}
```

## 8. 확장 및 커뮤니티
### 8.1 새로운 발견 알고리즘 추가 절차
1. `plugins/` 디렉터리 아래에 알고리즘 패키지 생성  
2. `BaseSolutionGenerator`를 구현하고 `__init__.py`에 등록  
3. `README.md`에 사용법·파라미터 설명 추가  
4. Pull Request 제출 → CI 자동 테스트 → 머지  

### 8.2 평가 지표·셀렉터 커스터마이징 팁
- **멀티 목표 최적화**: `selector.strategy = "pareto"` 로 설정하고 `weights` 파라미터 지정  
- **실시간 모니터링**: Prometheus exporter를 활성화하고 Grafana 대시보드에 메트릭 추가  

### 8.3 기여 가이드라인
- **코드 스타일**: `black`, `isort`, `flake8` 준수  
- **테스트**: `pytest` 기반 최소 80 % 커버리지 유지  
- **리뷰**: 최소 2명의 리뷰어 승인 필요
