## 1. 소개 및 중요성
Claude AI를 코딩, 글쓰기, 자동화 등에 활용하고 있다면 기본 설정만으로는 최적의 성능을 끌어내기 어렵습니다. `.claude/` 폴더는 Claude를 **맞춤형 생산성 엔진**으로 변환시키는 설정 레이어이며, 여기서 모델, 온도, 툴, 시스템 프롬프트 등을 정의합니다. Hacker News 토론에서도 개발자들이 이 레이어의 중요성을 점점 인식하고 있다는 점이 확인됩니다 [출처](https://euno.news/posts/ko/anatomy-of-the-claude-folder-a-deep-dive-into-clau-9f5de6).

## 2. `.claude/` 폴더 위치 및 전체 구조
- 기본 경로: `~/.claude/` (홈 디렉터리 아래)  
- 주요 서브디렉터리  
  - `config.json` – 핵심 설정 파일  
  - `tools/` – 커스텀 툴킷  
  - `prompts/` – 재사용 가능한 프롬프트 템플릿  
  - `history/` – 대화 기록·메모리  
  - `cache/` – 응답 캐시  

전체 트리 예시  

```
~/.claude/
├─ config.json
├─ tools/
├─ prompts/
├─ history/
└─ cache/
```  

## 3. `config.json` – 핵심 설정 파일
`config.json`은 Claude AI의 동작 방식을 정의하는 **두뇌** 역할을 합니다. 예시는 다음과 같습니다 [출처](https://euno.news/posts/ko/anatomy-of-the-claude-folder-a-deep-dive-into-clau-9f5de6).

```
{
  "model": "claude-3-5-sonnet-20241022",
  "temperature": 0.7,
  "max_tokens": 4096,
  "tools": ["code_interpreter", "file_search"],
  "system_prompt": "You are a senior developer focused on clean code."
}
```

| 설정 | 설명 |
|------|------|
| `model` | 사용할 Claude 모델 (버전 관리 가능) |
| `temperature` | 창의성 수준 (0.0 = 결정론적, 1.0 = 창의적) |
| `max_tokens` | 응답 길이 제한 |
| `tools` | 활성화할 툴 목록 |
| `system_prompt` | AI의 성격·역할 정의 |

## 4. `tools/` – 커스텀 툴킷
`tools/` 디렉터리는 Claude의 기능을 **플러그인** 형태로 확장합니다. 각 툴은 최소 세 파일을 포함합니다.

### 툴 구조 예시
```
~/.claude/tools/
├─ code_review/
│  ├─ manifest.json
│  ├─ handler.py
│  └─ README.md
├─ file_operations/
│  ├─ manifest.json
│  ├─ handler.py
│  └─ README.md
└─ web_search/
   ├─ manifest.json
   ├─ handler.py
   └─ README.md
```

### `manifest.json` 예시
```
{
  "name": "code_review",
  "version": "1.0.0",
  "description": "Automated code review with best practices",
  "entry_point": "handler.py",
  "permissions": ["read_files", "write_files"]
}
```
* `name` – 툴 식별자  
* `version` – 버전 관리  
* `description` – 기능 요약  
* `entry_point` – 실행 파일 (보통 Python)  
* `permissions` – 파일 접근 등 권한 선언  

툴은 `config.json`의 `tools` 배열에 이름을 추가함으로써 활성화됩니다.

## 5. `prompts/` – 재사용 가능한 프롬프트 템플릿
프롬프트 템플릿은 `.md` 파일 형태로 저장되며, 변수 삽입(`{{variable}}`)을 지원합니다. 이는 일관된 작업 흐름과 시간 절약을 가능하게 합니다 [출처](https://euno.news/posts/ko/anatomy-of-the-claude-folder-a-deep-dive-into-clau-9f5de6).

### 디렉터리 레이아웃
```
~/.claude/prompts/
├─ code_review.md
├─ bug_fix.md
├─ feature_implementation.md
└─ documentation.md
```

### `code_review.md` 예시
```
# Code Review Template
Review the following code for:
1. Security vulnerabilities
2. Performance issues
3. Code style violations
4. Edge cases

Code:
{{code}}

Provide:
- Summary of issues
- Line‑by‑line feedback
- Suggested fixes
```

## 6. `history/` – 대화 기록 및 메모리
Claude와의 상호작용은 `history/`에 세션별 JSON 파일로 저장됩니다. 이는 컨텍스트 유지와 향후 학습에 활용됩니다 [출처](https://euno.news/posts/ko/anatomy-of-the-claude-folder-a-deep-dive-into-clau-9f5de6).

### 저장 구조 예시
```
~/.claude/history/
├─ 2026-03-28/
│  ├─ session_001.json
│  ├─ session_002.json
│  └─ session_003.json
└─ 2026-03-27/
   ├─ session_001.json
   └─ session_002.json
```

* **세션 간 컨텍스트 유지** – 이전 대화 내용이 자동으로 포함됩니다.  
* **학습 활용** – 과거 상호작용을 분석해 프롬프트 품질을 개선할 수 있습니다.  

## 7. `cache/` – 응답 캐시와 성능 최적화
`cache/`는 자주 사용되는 프롬프트와 API 응답을 저장해 **지연 시간 감소**와 **비용 절감**을 도와줍니다 [출처](https://euno.news/posts/ko/anatomy-of-the-claude-folder-a-deep-dive-into-clau-9f5de6).

### 캐시 대상
- 프롬프트 문자열  
- Claude API 응답 (JSON)  

### 관리 방법
- **TTL(시간‑to‑live)** 설정을 통해 일정 기간 후 자동 삭제  
- 필요 시 수동으로 `cache/` 디렉터리를 정리  

## 8. 설정 적용 및 실행 흐름
1. **초기 설정** – `~/.claude/`에 `config.json`·`tools/`·`prompts/` 등을 배치.  
2. **Claude 실행** – CLI 혹은 SDK가 시작될 때 `config.json`을 읽어 모델·온도·툴을 로드.  
3. **툴·프롬프트 연동** – 사용자가 특정 툴을 호출하면 `manifest.json` → `handler.py` → 실행, 프롬프트 템플릿은 `prompts/`에서 로드.  
4. **히스토리·캐시 기록** – 각 세션 결과는 `history/`와 `cache/`에 저장되어 다음 호출 시 재사용 가능.  

## 9. 베스트 프랙티스
| 영역 | 권고사항 |
|------|----------|
| `config.json` | 모델은 작업 특성에 맞게 선택하고, 온도는 창의성 요구에 따라 0.2‑0.8 사이에서 조정. `max_tokens`는 예상 응답 길이에 맞게 설정. |
| 툴 개발 | `manifest.json`에 최소 권한만 선언하고, 외부 의존성은 `requirements.txt` 등으로 명시. 보안 검토 후 배포. |
| 프롬프트 관리 | 템플릿 파일은 Git 등 버전 관리 시스템에 보관하고, 변수 표준화를 위해 `{{variable}}` 형식을 일관적으로 사용. |
| 히스토리·캐시 | 주기적인 정리 스크립트를 설정하고, 중요한 세션은 별도 백업 디렉터리로 복제. |

## 10. 트러블슈팅 가이드
- **설정 파일 파싱 오류**  
  - JSON 형식 오류(쉼표 누락, 따옴표 불일치) 확인.  
  - `config.json`을 편집 후 `jq` 등으로 검증.  
- **툴 로드 실패**  
  - `manifest.json`의 `entry_point` 경로와 파일 권한 확인.  
  - 로그(`~/.claude/logs/` 혹은 콘솔)에서 `PermissionError` 메시지 탐색.  
- **프롬프트 템플릿 적용 오류**  
  - 템플릿 파일 인코딩이 UTF‑8인지 확인.  
  - 변수(`{{code}}` 등) 누락 시 기본값을 제공하거나 오류를 명시.  
- **캐시/히스토리 손상**  
  - 손상된 JSON 파일은 백업에서 복원하거나 `jq --compact-output`으로 재생성.  
  - 전체 캐시 삭제는 `rm -rf ~/.claude/cache/*` 로 수행 가능 (재생성 시 자동).  

## 11. 향후 확장 방향
- **플러그인 아키텍처**: Docker 컨테이너 기반 툴 배포, 언어 독립적 `entry_point` 지원.  
- **멀티모델 지원**: `config.json`에 모델 리스트를 정의하고, 세션별 동적 전환 로직 구현.  
- **커뮤니티 템플릿·툴 레포**: GitHub 조직(예: `claude-tools`)에 공개 템플릿·툴을 공유하고, `README.md`에 설치 가이드 제공.
