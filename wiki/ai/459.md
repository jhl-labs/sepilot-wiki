## 1. 서론
### AI 코딩 어시스턴트 활용 현황
Claude, Cursor, Codex, Gemini CLI 등 최신 LLM 기반 코딩 어시스턴트는 개발자의 생산성을 크게 끌어올리고 있습니다. 하지만 **“지난 세션을 어떻게 찾을까?”** 라는 질문이 반복됩니다.  

### 개발자들이 겪는 “지난 세션 찾기” 문제 정의
- 터미널 히스토리: 명령어만 남고 프롬프트·추론 과정은 사라짐  
- Git 로그: 커밋 메시지만 보이고, 대화 흐름·중간 코드 상태는 기록되지 않음  
- 브라우저 히스토리: `claude.ai` 같은 URL이 남지만, 실제 대화 내용은 포함되지 않음  

### 문서 목적 및 기대 효과
본 가이드는 **Mantra** 라는 세션 기록·검색 도구의 설계·구현·운영 방안을 제시함으로써  
- 재구성에 소요되는 시간을 최소화하고  
- 팀 전체 생산성을 회복하며  
- 민감 데이터 유출 위험을 방지하고자 합니다.

## 2. 문제 진단
### 기존 로그·git·브라우저 히스토리의 한계
| 자료 | 제공 정보 | 한계 |
|------|-----------|------|
| 터미널 히스토리 | 실행된 명령어 | 프롬프트·AI 응답·코드 스냅샷 없음 |
| Git 로그 | 커밋 메시지·변경 파일 | 중간 단계 코드·대화 흐름 비가시화 |
| 브라우저 히스토리 | URL 방문 기록 | 대화 내용·터미널 I/O 미포함 |

### 세션 재구성에 소요되는 시간·비용
- **주당 3–4회** 이전 세션을 참조하고자 함  
- **각 회당 약 20분**을 컨텍스트 재구성에 소비 (Dev.to 기사)  
- **주당 1–1.5시간**을 이미 해결된 문제를 다시 프롬프트하는 데 낭비  

### 팀 규모에 따른 생산성 손실 추정
5인 개발팀 기준으로 **주당 전체 근무일 1일**을 잃는 셈(Dev.to 기사).

## 4. 솔루션 개요 – Mantra
### 제품 개념 및 목표
Mantra는 **무마찰 기록**·**빠른 재생**·**내장 보안**을 핵심 가치로 하는 AI 코딩 세션 히스토리 뷰어입니다.  

### 전체 흐름
1. **세션 기록** – 클라이언트‑사이드 레코더가 WebSocket, 터미널, 프롬프트를 가로채 저장  
2. **저장** – 로컬 파일(예: JSONL) 혹은 클라우드 SQLite DB에 압축 저장  
3. **인덱싱** – 검색 엔진이 키워드·파일·툴 메타데이터를 인덱싱  
4. **재생** – UI에서 원하는 시점·파일·터미널 출력을 선택해 즉시 재현  

### 주요 특징 요약
- **마찰 없는 백그라운드 레코딩** (2분 셋업)  
- **시점 기반 빠른 재생** (전체 재생이 아니라 특정 메시지·코드만)  
- **자동 민감 데이터 탐지·마스킹** (API 키·비밀번호 등)  

---

## 5. 아키텍처 설계
### 클라이언트‑사이드 레코더 모듈
- **WebSocket 인터셉터**: Claude·Cursor 등 LLM 서비스와의 실시간 통신 가로채기  
- **터미널 프록시**: `script` 혹은 `asciinema`와 유사한 방식으로 입출력 스트림 캡처  
- **프롬프트 캡처**: UI 입력창(예: VS Code 확장)에서 전송 전 프롬프트를 기록  

### 중앙 저장소(로컬/클라우드) 구조
- **파일 포맷**: 라인 단위 JSONL (각 라인 = 이벤트 객체) – 가독성·스트리밍에 유리  
- **압축 전략**: gzip 압축을 기본 적용 (공식 문서: <https://gzip.org>)  

### 인덱싱·검색 엔진 설계
- **SQLite FTS5**를 활용한 전체 텍스트 검색 및 메타데이터 필터링 (공식 문서: <https://www.sqlite.org/fts5.html>)  
- **키워드·파일·툴** 별 별도 인덱스 테이블 구축  

### 보안 모듈
- **키워드 필터링**: 정규식 기반 민감 문자열 탐지 (예: `AKIA[0-9A-Z]{16}`)  
- **암호화 옵션**: 저장 파일을 AES‑256으로 암호화 (OpenSSL 라이브러리 사용)

## 6. 구현 상세
### 레코딩
- **WebSocket 인터셉트**: 브라우저 확장 혹은 프록시 서버가 `ws.send`/`ws.onmessage`를 가로채 JSON 형태로 저장  
- **터미널 프록시**: `script -q -f` 명령어와 유사하게 PTY를 래핑해 입출력 스트림을 파일에 기록  
- **프롬프트 캡처**: IDE 플러그인에서 `onDidChangeTextDocument` 이벤트를 활용  

### 스토리지
- **JSONL**: `{ "timestamp": "...", "type": "prompt|response|terminal", "content": "...", "metadata": {...} }` 형태  
- **SQLite**: 세션 메타데이터 테이블(`sessions`), 이벤트 테이블(`events`)을 정의하고 FTS5 인덱스 적용  

### 검색 UI
- **필터**: 파일명, 사용 툴, 날짜 범위 슬라이더 제공  
- **키워드**: 실시간 자동완성 및 하이라이트 기능 구현  

### 재생 엔진
- **단계별 코드 복원**: 이벤트 순서대로 `apply_patch` 방식으로 파일 상태 재구성  
- **터미널 복원**: `asciinema` 포맷을 활용해 타임스탬프 기반 재생

## 7. 보안 및 프라이버시 고려사항
1. **API 키·자격증명 자동 탐지**  
   - 정규식 및 머신러닝 기반 민감 데이터 탐지 로직 적용 (추가 조사가 필요합니다)  
2. **민감 데이터 마스킹·삭제 정책**  
   - 탐지된 문자열을 `***MASKED***` 로 대체하고, 원본 파일에서 영구 삭제 옵션 제공  
3. **저장소 접근 제어 및 암호화 옵션**  
   - 파일 시스템 ACL, SQLite 암호화, 혹은 외부 KMS 연동 가능

## 8. 사용 가이드
### 설치·설정 (2분 셋업)
1. Mantra 바이너리(또는 npm 패키지) 다운로드  
2. `mantra init` 명령어로 기본 설정 파일(`mantra.yaml`) 생성  
3. 레코더 플러그인(브라우저 확장·VS Code) 설치 후 API 토큰 입력  

### 세션 기록 시작·중지
- `mantra start` 로 현재 터미널·WebSocket 레코딩 시작  
- `mantra stop` 로 기록 종료 및 파일 저장  

### 검색·재생 워크플로우
1. UI에서 날짜·키워드·파일 필터 지정  
2. 결과 리스트에서 원하는 시점 클릭 → 코드 스냅샷·터미널 출력 동시 표시  
3. `Play` 버튼으로 해당 시점부터 재생하거나, `Export` 로 별도 파일 저장  

### 팀 협업을 위한 공유·권한 관리
- 저장소를 공유 폴더 혹은 S3 버킷에 배포하고, IAM 정책으로 읽기/쓰기 권한 제어  
- 세션 메타데이터에 팀 태그를 추가해 UI에서 팀별 필터링 가능
