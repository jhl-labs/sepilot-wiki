## 1. 개요 및 배경
- **온‑디바이스 AI 추론의 필요성**  
  모바일 환경에서 클라우드 의존도를 낮추고 실시간 응답성을 확보하기 위해 모델을 디바이스에 직접 배치하는 추세가 확대되고 있습니다. 이는 네트워크 지연·연결 불안정 문제를 해소하고, 사용자 데이터가 디바이스를 떠나지 않도록 함으로써 프라이버시 보호에도 기여합니다.  

- **Google AI Edge Gallery 소개**  
  Google AI Edge Gallery는 Android·iOS 디바이스에서 Edge AI 모델을 손쉽게 실행할 수 있도록 지원하는 플랫폼입니다. 개발자는 Gallery 앱을 통해 모델을 등록·테스트하고, 온‑디바이스 함수 호출(on‑device function calling) 기능을 활용해 행동‑중심 AI 에이전트를 구현할 수 있습니다.  

- **클라우드‑중심 모델과의 차별점**  
  기존 클라우드‑중심 모델은 텍스트 생성·질의응답 등에 초점을 맞추지만, FunctionGemma와 같은 온‑디바이스 모델은 **함수 호출**을 직접 트리거하여 디바이스 하드웨어 제어나 로컬 로직 실행 등 행동을 즉시 수행합니다. 이는 지연 시간 감소와 오프라인 사용성을 동시에 제공합니다.  

## 2. FunctionGemma 모델 소개
- **모델 아키텍처 및 파라미터 규모**  
  FunctionGemma는 **270M 파라미터**를 보유한 특수 목적 모델로, 온‑디바이스 함수 호출을 효율적으로 수행하도록 설계되었습니다.  

- **행동‑중심 AI를 위한 설계 목표**  
  - 텍스트 입력 → 함수 매핑 자동화  
  - 모바일 하드웨어에 최적화된 연산 경량화  
  - 오프라인에서도 일관된 응답 제공  

- **지원되는 입력·출력 형식 및 API 개요**  
  - 입력: 자연어 텍스트 (예: “내일 회의 일정 추가해줘”)  
  - 출력: 사전 정의된 함수 시그니처와 매개변수 (예: `addEvent(date, title)`)  
  - API는 Google AI Edge Gallery 앱 내에서 모델을 로드하고, `invokeFunction()` 형태의 호출 인터페이스를 제공합니다.  

> 자세한 모델 스펙은 Google Developers Blog의 발표 자료를 참고하십시오. [출처](https://euno.news/posts/ko/on-device-function-calling-in-google-ai-edge-galle-ae4c94)

## 3. LiteRT‑LM을 활용한 온‑디바이스 실행
- **LiteRT‑LM 개념 및 역할**  
  LiteRT‑LM은 Edge 디바이스용 경량 런타임으로, FunctionGemma와 같은 대규모 언어 모델을 메모리·연산 효율적으로 실행하도록 지원합니다.  

- **Edge 디바이스(Android/iOS)에서의 배포 흐름**  
  1. **모델 변환** – FunctionGemma 모델 파일(.tflite 등)을 LiteRT‑LM 포맷으로 변환.  
  2. **앱에 포함** – 변환된 모델을 Android APK 또는 iOS IPA에 포함.  
  3. **런타임 초기화** – 앱 실행 시 LiteRT‑LM 엔진을 초기화하고 모델을 로드.  
  4. **함수 호출** – 사용자의 텍스트 입력을 모델에 전달하고, 반환된 함수 시그니처를 기반으로 로컬 로직을 실행.  

- **메모리·연산 최적화 기법**  
  - **양자화(Quantization)**: 8‑bit 정밀도로 변환하여 메모리 사용량을 절감.  
  - **연산 그래프 최적화**: 불필요한 연산 제거 및 연산 순서 재배치.  

> 현재 공개된 구체적인 최적화 파라미터는 없으며, 실제 디바이스에 적용 시 추가 조사가 필요합니다.

## 4. Google AI Edge Gallery와 통합 방법
- **Gallery 앱에 FunctionGemma 모델 등록 절차**  
  1. Google AI Edge Gallery 앱을 설치하고 실행.  
  2. “Add Model” 메뉴에서 변환된 LiteRT‑LM 모델 파일을 선택.  
  3. 모델 메타데이터(이름, 버전, 입력/출력 스키마)를 입력하고 “Register” 클릭.  

- **Android와 iOS 각각의 통합 단계**  

  **Android**  
  - `build.gradle`에 LiteRT‑LM 의존성 추가.  
  - `assets/` 폴더에 모델 파일 배치.  
  - `EdgeModelLoader` 클래스를 사용해 모델 로드 및 초기화.  

  **iOS**  
  - `Podfile`에 LiteRT‑LM CocoaPods 추가.  
  - Xcode 프로젝트에 모델 파일을 포함하고, `EdgeModelManager`를 통해 로드.  

- **샘플 프로젝트 구조 및 빌드 설정**  
  ```
  /app
    /src/main
      /java/com/example/edge
        EdgeModelLoader.java
        FunctionInvoker.java
    /assets
      functiongemma.litertlm
  /ios
    EdgeModelManager.swift
    FunctionInvoker.swift
    Resources/functiongemma.litertlm
  ```  

> 위 경로와 파일명은 예시이며, 실제 프로젝트에 맞게 조정해야 합니다.

## 5. 샘플 코드 및 사용 예시
> 아래 코드는 **코드 블록 없이** 들여쓰기로 표시된 예시입니다. 실제 프로젝트에 적용 시 적절히 수정하십시오.

### 기본 호출 예제 (텍스트 → 함수)
    // Android Kotlin 예시
    val loader = EdgeModelLoader(context)
    val model = loader.loadModel("functiongemma.litertlm")
    val result = model.invokeFunction("내일 10시에 회의 잡아줘")
    // result 예시: addEvent(date="2026-03-11T10:00", title="회의")

### 일정 관리 시나리오 구현
    // iOS Swift 예시
    let manager = EdgeModelManager()
    let model = manager.loadModel(named: "functiongemma")
    if let function = model.invokeFunction("다음 주 월요일에 점심 약속 추가해") {
        // function 은 사전 정의된 addEvent 함수 시그니처
        CalendarService.shared.addEvent(date: function["date"], title: function["title"])
    }

### 하드웨어 제어 및 Tiny Garden 데모 코드
    // Android Java 예시
    String userInput = "조명 켜줘";
    Map<String, String> fn = model.invokeFunction(userInput);
    if ("toggleLight".equals(fn.get("name"))) {
        HardwareController.toggleLight(true);
    }

> 실제 함수 시그니처와 매개변수는 FunctionGemma 모델에 정의된 대로 다를 수 있습니다.  

## 6. 성능 및 지연 시간 벤치마크
- **주요 모바일 칩셋별 추론 시간**  
  현재 Google에서 공개한 구체적인 추론 시간 데이터는 없습니다. 따라서 각 디바이스별 실측이 필요합니다.  

- **클라우드 호출 대비 지연 감소 비율**  
  온‑디바이스 실행은 네트워크 왕복 지연을 제거하므로, 일반적인 클라우드 기반 LLM 호출 대비 **수십 배** 빠른 응답이 기대됩니다. 정확한 비율은 실제 워크로드와 디바이스 사양에 따라 달라집니다.  

- **배터리·전력 소비 분석**  
  LiteRT‑LM의 양자화 및 최적화 덕분에 전력 소모가 최소화되지만, 정량적인 전력 프로파일은 공개되지 않았습니다. 배터리 영향 평가를 위해서는 장시간 실행 테스트가 필요합니다.  

> 위 항목들은 현재 공개된 자료가 없으므로 **추가 조사가 필요합니다**.

## 7. 보안 및 프라이버시 고려사항
- **데이터 노출 최소화 방안**  
  - 모든 입력·출력 데이터는 디바이스 내에서 처리되며, 네트워크 전송이 발생하지 않습니다.  
  - 필요 시 로컬 암호화 저장소(예: Android Keystore, iOS Keychain)를 활용해 민감 데이터 보호가 가능합니다.  

- **모델 무결성 검증 및 서명 검증 절차**  
  - Google AI Edge Gallery는 모델 파일에 디지털 서명을 부여합니다. 앱이 모델을 로드할 때 서명 검증을 수행하여 변조 여부를 확인합니다.  

- **사용자 권한 관리 및 정책 적용 방법**  
  - 함수 호출이 하드웨어 제어(예: 카메라, 마이크, 조명)와 연결될 경우, 해당 권한을 Android Manifest 또는 iOS Info.plist에 명시하고, 런타임에 사용자 동의를 받아야 합니다.  

## 8. FAQ
**Q1. 지원되는 디바이스 및 OS 버전은?**  
A. 현재 Google AI Edge Gallery는 Android 8.0 이상 및 iOS 13 이상을 지원합니다. 구체적인 디바이스 리스트는 공식 문서에 명시되어 있지 않으므로, 최신 스마트폰 모델에서 테스트하는 것이 권장됩니다.  

**Q2. 모델 업데이트와 호환성은 어떻게 유지하나요?**  
A. Gallery 앱을 통해 새로운 모델 파일을 등록하면 기존 앱은 자동으로 최신 모델을 로드합니다. 다만, 모델 인터페이스(함수 시그니처)가 변경될 경우 앱 코드도 함께 업데이트해야 합니다.  

**Q3. 오류 발생 시 디버깅 팁 및 로그 확인 방법은?**  
A. LiteRT‑LM은 표준 Android Logcat / iOS Console 로그에 초기화·추론 단계별 메시지를 출력합니다. `EdgeModelLoader` 혹은 `EdgeModelManager` 클래스에서 제공하는 콜백을 활용해 오류 코드를 확인할 수 있습니다.  

**Q4. 라이선스 및 사용 제한 사항은?**  
A. FunctionGemma와 LiteRT‑LM은 Google의 개발자 라이선스 하에 제공됩니다. 상업적 이용 시 Google Cloud Terms of Service 및 해당 모델의 사용 정책을 준수해야 합니다. 자세한 내용은 Google Developers Blog 및 해당 모델 배포 페이지를 참고하십시오.  

**Q5. 온‑디바이스 함수 호출이 클라우드와 혼합 사용이 가능한가요?**  
A. 네, 필요에 따라 온‑디바이스와 클라우드 기반 LLM을 병행할 수 있습니다. 예를 들어, 복잡한 질의는 클라우드에 전달하고, 단순한 제어 명령은 온‑디바이스 FunctionGemma가 처리하도록 설계할 수 있습니다.
