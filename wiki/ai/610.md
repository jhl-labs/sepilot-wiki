## 1. 문서 개요
### 목적 및 대상 독자
이 가이드는 **n8n**을 **MCP (Multi‑Channel Platform) 허브**로 전환하여 외부 AI 에이전트(예: Claude Desktop, Cursor, VS Code) 가 워크플로를 직접 호출하도록 구성하고자 하는 개발자·엔지니어·플랫폼 운영자를 대상으로 합니다.  

### 핵심 개념 정의
- **MCP**: 여러 AI·툴이 서로 호출·연동할 수 있게 하는 양방향 인터페이스.  
- **AI‑callable 워크플로**: n8n 워크플로를 HTTP 엔드포인트로 노출해 AI 에이전트가 마치 “함수”처럼 호출할 수 있는 형태.  

### 이 가이드가 제공하는 가치
- n8n을 **양방향 MCP 허브**로 활용하는 전체 흐름 이해  
- **MCP Server Trigger**와 **MCP Client Tool** 노드 설정 방법  
- 자체 호스팅·n8n Cloud 선택 시 고려사항 및 비용 구조  
- 보안·신뢰성 확보 방안 및 실전 사용 사례 제공

## 3. 아키텍처 및 흐름
### 전체 데이터 흐름도 (텍스트 설명)

```
외부 AI 에이전트
   │
   ▼
MCP Server Trigger (n8n)
   │   (HTTP /mcp/<workflow-id>)
   ▼
워크플로 실행
   │
   ├─> MCP Client Tool (GitHub, Slack, DB 등)
   │
   ▼
외부 MCP 서버 / 서비스
   │
   ▼
결과 반환 → AI 에이전트
```

### 핵심 컴포넌트
- **MCP Server Trigger**: 워크플로를 `/mcp/<workflow-id>` 엔드포인트로 노출.  
- **MCP Client Tool**: 워크플로 내에서 외부 도구(GitHub, Slack, DB 등)를 호출하도록 구성.  

### 오류 처리 및 HITL
n8n은 **오류 처리와 Human‑In‑The‑Loop (HITL) 제어** 를 내장하고 있어, 워크플로 중 오류 발생 시 자동 재시도·알림·수동 개입을 설계할 수 있습니다 (출처: Dev.to).

## 4. 설치 및 호스팅 옵션
| 옵션 | 특징 | 비용 |
|------|------|------|
| **자체 호스팅** | 무료(오픈소스) + 인프라 비용 | 직접 관리 |
| **n8n Cloud** | 관리형 서비스, 실행 무제한 | €24 – €800 / mo (무료 티어 포함) (출처: Dev.to) |

### 최소 시스템 요구사항
- Node.js 18+  
- PostgreSQL/MySQL 등 지원되는 DB  
- 공개 URL을 가진 실행 중인 인스턴스 (STDIO 전송 없음) (출처: Dev.to)

### 권장 사양
- CPU 2 vCPU 이상, 메모리 4 GB 이상 (자체 호스팅 시)  
- TLS 인증서 적용된 도메인

## 5. MCP Server Trigger 설정
1. **노드 추가**: 워크플로 편집 화면 → “+” → “MCP Server Trigger” 선택.  
2. **기본 설정**:  
   - **Endpoint**: `/mcp/<workflow-id>` 형태 자동 생성.  
   - **Method**: POST (기본)  
3. **리버스 프록시 구성** (예: Nginx)  
   - `proxy_buffering off;` 로 버퍼링 비활성화 (출처: Dev.to)  
   - `proxy_set_header Host $host;` 등 기본 헤더 전달 설정  
4. **다중 복제 제한**  
   - 모든 `/mcp*` 요청은 **단일 웹훅 복제본** 으로 라우팅해야 함. 다중 복제 설정은 MCP 기능을 깨뜨림 (출처: Dev.to).

## 6. MCP Client Tool 활용법
| 지원 도구 | 활용 예시 |
|-----------|----------|
| **GitHub** | PR 생성, 이슈 조회 |
| **Slack** | 채널에 메시지 전송, 파일 업로드 |
| **데이터베이스** | SQL 쿼리 실행, 레코드 삽입/조회 |

### 노드 구성 예시 (인라인)
- **Node**: `MCP Client Tool`  
- **Tool**: `GitHub` → `Action: Create Issue`  
- **Parameters**: `title: {{ $json.title }}`, `body: {{ $json.body }}`  

### AI 에이전트와 연동 시나리오
1. AI 에이전트가 `GET /mcp/get_customer_360` 호출  
2. n8n 워크플로가 CRM → DB 조회 → 결과를 **MCP Client Tool** 로 Slack 알림 전송  
3. 최종 요약을 AI 에이전트에 반환

## 7. 보안 및 신뢰성 고려 사항
- **인증·인가**: API 키 또는 OAuth2 토큰을 헤더(`Authorization: Bearer <token>`) 로 전달.  
- **HTTPS**: 반드시 TLS 적용된 도메인 사용.  
- **프록시 버퍼링 비활성화**: Nginx 등 리버스 프록시에서 `proxy_buffering off;` 설정 필요 (출처: Dev.to).  
- **트리거 신뢰성**: 커뮤니티 보고에 따르면 첫 사용 후 트리거가 중단될 수 있음. 지속적인 모니터링과 재시작 스크립트 적용 권장 (출처: Dev.to).

## 8. 실전 사용 사례
### 1) 고객 360° 조회 워크플로
1. **MCP Server Trigger**: `/mcp/get_customer_360`  
2. **CRM 노드**: 고객 기본 정보 조회  
3. **DB 노드**: 티켓 데이터와 교차 참조  
4. **MCP Client Tool (Slack)**: 요약을 담당 팀 채널에 전송  
5. **응답**: 요약 JSON을 AI 에이전트에 반환  

### 2) 티켓 시스템 연동 및 자동 요약
- AI 에이전트가 새로운 티켓 ID 로 호출 → n8n이 해당 티켓 상세를 가져와 **OpenAI** 요약 API 호출 후 결과 반환.  

### 3) 복합 비즈니스 로직 노출 패턴
- 여러 외부 MCP 서버(GitHub, Jira, 내부 DB)를 **MCP Client Tool** 로 연계, AI 에이전트가 단일 엔드포인트로 복합 로직을 실행하도록 설계.

## 9. 비용·라이선스 상세 분석
- **실행당 비용 모델**: n8n은 **전체 워크플로 실행** 단위로 과금 (작업 수와 무관) (출처: Dev.to).  
- **무료 티어**: 자체 호스팅 시 무제한 실행 가능 (오픈소스)  
- **Enterprise 플랜**: 상업적 재배포 및 고급 지원 제공 (Fair‑code 라이선스)  

### 비용 최적화 전략
- **워크플로 합치기**: 불필요한 작은 실행을 하나의 워크플로로 통합  
- **트리거 재사용**: 동일 엔드포인트에 여러 로직을 매핑해 호출 횟수 최소화

## 11. 베스트 프랙티스 및 팁
- **성능 최적화**: 가능한 한 **단일 워크플로**에 노드를 집중하고, 불필요한 반복 호출을 피함.  
- **다중 MCP 서버 충돌 방지**: 각 MCP Client Tool 에는 고유 **namespace** 를 지정하고, 동일 엔드포인트에 대한 라우팅을 명확히 설정.  
- **로그·모니터링**: n8n 내장 **Execution Log** 와 외부 **Prometheus/Grafana** 연동을 통해 트리거 성공률을 실시간 감시.

## 12. 문제 해결 가이드
| 오류 유형 | 원인 | 해결 방법 |
|-----------|------|-----------|
| 트리거 중단 | 리버스 프록시 버퍼링 활성화 | `proxy_buffering off;` 적용 (출처: Dev.to) |
| 404 Not Found | 엔드포인트 URL 오타 또는 복제 설정 오류 | `/mcp/<workflow-id>` 형식 확인, 복제 제한 검토 |
| 인증 실패 | API 키 누락 또는 잘못된 포맷 | 헤더에 `Authorization: Bearer <key>` 정확히 전달 |
| 응답 지연 | 외부 도구 호출 시간 초과 | 워크플로 내 **Timeout** 설정 조정, 비동기 처리 활용 |

**디버깅 팁**  
- 실행 로그에서 **Node ID** 로 문제 노드 식별  
- `curl -v https://your-n8n-instance.com/mcp/<id>` 로 직접 호출 테스트
