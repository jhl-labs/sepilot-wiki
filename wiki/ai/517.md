## 소개
AI 서비스 사용 시 발생하는 **“청구서 서프라이즈”** 문제를 해결하기 위한 오픈‑소스 도구 **TokenWatch** 를 안내합니다.  
대상 독자는 **개발자, 팀 리더, 비용 관리 담당자** 로, AI 모델 호출 비용을 투명하게 파악하고 예산 초과를 방지하고자 하는 모든 조직에 적용할 수 있습니다.

AI 비용 관리가 중요한 이유  
- 모델별 토큰당 가격 차이가 크고,  
- 프로젝트 규모가 커질수록 작은 비용 누적이 큰 예산 초과로 이어지기 때문입니다.  

TokenWatch는 **로컬 전용, 프라이버시 보장, 자동 알림** 기능을 제공하여 위 문제를 해결합니다.

## “청구서 서프라이즈” 문제 정의
- **비용 불투명성**: 실시간 사용량이 보이지 않아 청구서가 도착했을 때야말로 총 지출을 알게 됩니다.  
- **모델·작업별 비용 식별 어려움**: 다양한 모델(OpenAI, Anthropic 등)과 작업(요약, 데이터 추출 등) 사이에서 비용이 많이 드는 부분을 파악하기 어렵습니다.  
- **기존 솔루션 한계**: 대부분 SaaS 기반 모니터링 툴은 클라우드 로그에 의존하고, 텔레메트리 수집으로 프라이버시 위험을 내포합니다.  

## TokenWatch 개요
- **오픈‑소스, MIT 라이선스** 로 자유롭게 사용·수정·배포 가능합니다.  
- **로컬 전용 설계**: 외부 API 키가 필요 없으며, 모든 데이터는 사용자의 머신에 있는 `.tokenwatch` 디렉터리에 저장됩니다. 텔레메트리를 전혀 수집하지 않아 **프라이버시와 자율성**을 보장합니다.  
- **지원 제공업체·모델**: 현재 10개 주요 제공업체와 41개 모델을 지원한다고 알려져 있으나, 구체적인 수치는 원문에 명시되지 않아 **출처 필요** 로 표시합니다.  

## 핵심 개념
### 청구서 서프라이즈 방지
- 실시간 사용량 추적을 통해 비용 초과를 사전에 감지합니다.  

### TokenWatch 주요 기능
| 기능 | 설명 |
|------|------|
| **세밀한 사용량 추적** | SDK 훅을 통한 자동 기록 및 `record_usage` API 로 수동 로그 기록 가능 |
| **예산 관리·알림** | 일/주/월 한도 설정 및 임계값 도달 시 로컬 알림 또는 콜백 실행 |
| **모델 비용 비교** | 실시간 가격 데이터를 기반으로 토큰당 비용을 즉시 계산 |
| **최적화 제안** | 고비용 모델을 저비용 대안으로 교체하거나 프롬프트 길이 최적화 등 구체적인 조언 제공 |
| **프라이버시 보장** | 모든 기록은 로컬 JSON 파일에 저장되고, 외부 전송이 전혀 없습니다 |

## 설치 및 초기 설정
- **시스템 요구사항**: Python 3.8 이상, pip 또는 conda 환경  
- **설치 명령**: `pip install tokenwatch` 혹은 `conda install -c conda-forge tokenwatch`  
- **디렉터리 구조**  
  ```
  프로젝트 루트
  ├─ .tokenwatch/      # 사용량 로그·설정 파일
  └─ tokenwatch_config.py   # 가격·예산 설정
  ```

## 구성 방법
- **가격 데이터 커스터마이징**: `PROVIDER_PRICING` 딕셔너리에 `{ "openai": { "gpt-4o-mini": 0.000015, ... } }` 형태로 모델별 토큰당 가격을 추가·수정합니다.  
- **예산·알림 파라미터**: `budget = { "daily": 10, "monthly": 300, "alert_at_percent": 80 }` 와 같이 설정 파일에 정의합니다.  
- **라벨·태그 정책**: 라벨은 문자열 리스트로 지정하고, 일관된 네이밍 규칙(예: `action:task`)을 유지하면 분석이 용이합니다.  

## 예시
### SDK 훅 적용
```python
import tokenwatch
tokenwatch.enable_openai_hook()      # OpenAI SDK 자동 추적
tokenwatch.enable_anthropic_hook()  # Anthropic SDK 자동 추적
```

### 수동 로그 기록
```python
tokenwatch.record_usage(
    provider="openai",
    model="gpt-4o-mini",
    prompt_tokens=120,
    completion_tokens=350,
    label="summarize article"
)
```

### 세션 기반 로그 (CI/CD 활용)
```python
tokenwatch.start_session()
# ... AI 호출 수행 ...
tokenwatch.end_session()
```

### 커스텀 제공업체·모델 추가
```python
tokenwatch.PROVIDER_PRICING["myprovider"] = { "my-model": 0.00002 }
```

### 플러그인 활용 예
```python
# plugins/my_plugin.py
def before_call(**kwargs):
    # 예: 프롬프트 길이 제한
    pass

tokenwatch.register_plugin("before_call", before_call)
```

## 베스트 프랙티스
- **라벨링 전략**: 기능 단위와 비즈니스 도메인을 조합해 라벨을 정의하면 비용 분석이 직관적입니다.  
- **분석 주기**: 최소 **주간** 단위로 비용 리포트를 검토하고, 급증 패턴이 보이면 알림 임계값을 조정합니다.  
- **알림 설정 포인트**: 월 예산 70 % 도달 시 알림, 90 % 도달 시 자동으로 저비용 모델 전환 스크립트를 트리거하도록 구성합니다.  
- **모델 선택 기준**: 동일 토큰 수 대비 **성능(정확도·응답 시간)** 과 **비용** 을 비교해 비핵심 작업은 저비용 모델(`gpt-4o-mini` 등)으로 전환합니다.  

## 제한 사항 및 고려 사항
- **지원되지 않는 플랫폼**: Windows 10 이하 및 일부 구형 Linux 배포판에서는 의존성 충돌이 발생할 수 있습니다.  
- **로컬 저장소 보안**: `.tokenwatch` 폴더는 민감한 프롬프트와 사용량 데이터를 포함하므로 파일 시스템 권한을 제한하고 정기적인 백업을 권장합니다.  
- **가격 데이터 업데이트**: 제공업체 정책에 따라 가격이 변동되므로 `tokenwatch update-pricing` 명령을 **주기적으로** 실행해 최신 데이터를 유지해야 합니다.  

## 기존 도구와 비교
| 항목 | SaaS 기반 비용 모니터링 툴 | TokenWatch |
|------|---------------------------|------------|
| **배포 형태** | 클라우드 서비스 (로그 전송 필요) | 로컬 전용, 오프라인 |
| **프라이버시** | 텔레메트리·프롬프트 전송 | 텔레메트리 없음, 데이터 로컬 저장 |
| **API 키 요구** | 필요 (서비스 연동) | 필요 없음 |
| **지원 모델** | 제한적 (주요 제공업체 중심) | 10개 제공업체·41개 모델 (2026‑02) **출처 필요** |
| **비용** | 구독료 발생 가능 | 무료 (MIT 라이선스) |
| **알림·예산** | 기본 알림 제공 | 세밀한 일/주/월·임계값 알림 지원 |
| **확장성** | 제한적 (플러그인 제한) | Python 플러그인·커스텀 모델 자유 추가 |

## 결론
TokenWatch는 **프라이버시를 보장하면서** AI 비용을 **실시간·세밀하게** 추적하고, **예산 초과를 사전에 방지**하는 강력한 로컬 도구입니다.  

- **조직**: 대규모 AI 프로젝트·멀티‑팀 환경에서 비용 가시성을 확보하고 예산 관리 프로세스를 자동화합니다.  
- **개인 개발자**: 실험 단계에서 비용 폭주를 방지하고 최적 모델 선택에 도움을 줍니다.
