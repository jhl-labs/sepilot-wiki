## 자동화 파이프라인
### 일일 다이제스트 흐름
1. **07:00 AM** – Hacker News와 GitHub Trending 페이지를 **표준 라이브러리**(`urllib`, `html.parser` 등)만 사용해 스크랩.  
2. 수집된 데이터를 **간단한 텍스트 파싱**으로 정제.  
3. **다이제스트**(Markdown) 파일 생성.  
4. **GitHub Pages** 레포에 커밋·푸시하여 공개.  

- **외부 API 키**나 서드파티 서비스 없이 순수 파이썬으로 구현.  
- 지난 주 전체 **성공적으로 실행**되었으며, 현재까지 오류 보고 없음【euno.news】.

## 교훈 및 베스트 프랙티스
1. **프라이버시 중심 도구**는 사용자 신뢰를 빠르게 확보한다.  
2. **표준 라이브러리만 활용한 자동화**는 의존성 관리와 보안 위험을 최소화한다.  
3. **채널 다변화**가 필수 – Twitter 정지 사례에서 보듯, 하나의 플랫폼에 과도하게 의존하면 리스크가 커진다.  
4. **제품·마케팅 실험**을 통한 빠른 피드백 루프가 첫 판매 달성에 핵심이다.
