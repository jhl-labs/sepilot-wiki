## 관련 저장소 (SEPilot Wiki)
- **URL**: https://github.com/jhl-labs/sepilot-wiki  
- **설명**: AI 에이전트 기반 자동화 위키 시스템으로, `/wiki` 폴더를 데이터 저장소로 활용하고 GitHub Issues와 AI를 통해 문서 생성·수정·유지보수를 자동화합니다.  
- **주요 기능**  
  - 저장소 기반 Wiki (GitHub Contents API)  
  - GitHub Pages 프론트엔드 (React + TypeScript + Vite)  
  - AI 기반 문서 작성 (Issue에 `request` 라벨)  
  - 협업 워크플로우 (Maintainer가 Issue 댓글로 피드백)  
  - Wiki Tree 자동 정비  
  - 자동 정보 수집 (cron 스케줄)  
- **기술 스택**  
  - **Framework**: React 18.3.1, TypeScript 5.x  
  - **Build Tool**: Vite 7.2.6  
  - **Runtime**: Bun 1.0+  
  - **State Management**: TanStack Query 5.x  
  - **Routing**: React Router 7.x  
  - **Styling**: CSS Variables + Custom CSS  
  - **Testing**: Vitest + Testing Library  
  - **CI/CD**: GitHub Actions (배포, Issue 처리, Wiki 정비, 시스템 정보 수집)  
  - **보안**: Path Traversal 방지, XSS 방지 (DOMPurify), CSP 헤더, Prompt Injection 방지  
- **프로젝트 구조**  
  ```
  sepilot-wiki/
  ├── .github/
  │   └── workflows/
  │       ├── deploy-pages.yml
  │       ├── issue-handler.yml
  │       ├── wiki-tree-maintainer.yml
  │       └── scheduled-collect.yml
  ├── wiki/                 # Wiki 문서 (Markdown)
  ├── src/
  │   ├── components/
  │   ├── pages/
  │   ├── hooks/
  │   ├── context/
  │   ├── types/
  │   ├── services/
  │   └── styles/
  ├── scripts/
  ├── CLAUDE.md
  ├── CHANGELOG.md
  └── README.md
  ```
- **설치 및 실행**  
  ```bash
  # 의존성 설치
  bun install   # 또는 npm install

  # 개발 서버 실행
  bun dev

  # 빌드
  bun run build

  # 테스트 실행
  bun test
  ```
- **기여 방법**  
  - Issue를 통해 기능 제안 또는 버그 리포트  
  - `request` 라벨을 달아 문서 작성 요청  
  - PR을 통한 직접 기여 (코드 스타일: ESLint + Prettier, TypeScript strict 모드)
