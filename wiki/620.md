## 1. 문서 개요
**목적** – Git MCP Server(이하 Git MCP)를 처음 접하는 개발자·운영자를 위해 핵심 기능, 통계, 보안 현황 및 사용 방법을 한눈에 정리한다.  
**대상 독자** – Git 기반 자동화·CI/CD 파이프라인을 설계·운영하는 엔지니어, 오픈소스 기여자, 보안 담당자.  

**Git MCP 정의** – Anthropic이 공식 레퍼런스로 제공하는, 로컬 Git 레포지토리 작업을 API 형태로 노출하는 경량 서버. Zero‑dependency 설계와 PyPI 배포가 특징이며, 현재 12개의 기본 도구만을 제공한다【※출처 확인 필요】.

## 3. 아키텍처 및 배포 모델
- **Zero‑dependency 설계** – API 키·계정이 필요 없으며, `--repository` 옵션에 로컬 경로만 지정하면 즉시 사용 가능.  
- **배포** – PyPI 패키지 `git-mcp-server` 로 제공. `pip install git-mcp-server` 로 설치 후 `git-mcp-server` 실행 파일이 생성된다.  
- **패키지 구조** – `git_mcp` 모듈 아래에 12개의 CLI 도구가 각각 독립 엔트리포인트로 등록되어 있다.

## 4. 제공 도구 상세 목록
| 도구 | 주요 기능 | 사용 예시 | 반환값(요약) |
|------|-----------|----------|--------------|
| `git_status` | 현재 워킹 디렉터리·스테이징 상태 출력 | `git_status --repository /path/to/repo` | 텍스트 형태의 파일 상태 리스트 |
| `git_diff_unstaged` | 워킹 디렉터리와 인덱스 간 차이점 표시 | `git_diff_unstaged --repository /path/to/repo` | `diff` 포맷 문자열 |
| `git_diff_staged` | 인덱스와 HEAD 간 차이점 표시 | `git_diff_staged --repository /path/to/repo` | `diff` 포맷 문자열 |
| `git_diff` | 지정 레퍼런스(HEAD, 브랜치 등)와 비교 | `git_diff --repository /path/to/repo --ref main` | `diff` 포맷 문자열 |
| `git_add` | 파일·디렉터리를 스테이징 영역에 추가 | `git_add --repository /path/to/repo --path src/` | 성공 시 `0`, 오류 시 비정상 종료 |
| `git_reset` | **전체** 리셋 – 인덱스를 HEAD와 동일하게 함 | `git_reset --repository /path/to/repo` | 성공 시 `0` |
| `git_commit` | 커밋 메시지와 옵션을 받아 커밋 수행 | `git_commit --repository /path/to/repo --message "Add feature X"` | 커밋 해시 문자열 |
| `git_log` | 로그 조회 – 날짜 범위 필터 가능 | `git_log --repository /path/to/repo --start_timestamp 2024-01-01T00:00:00Z --end_timestamp 2024-01-31T23:59:59Z` | JSON 배열(커밋 메타데이터) |
| `git_show` | 객체(커밋, 블롭 등) 상세 내용 출력 | `git_show --repository /path/to/repo --object <hash>` | 텍스트/바이너리 내용 |
| `git_create_branch` | 새 브랜치 생성 | `git_create_branch --repository /path/to/repo --branch feature-x` | 성공 시 `0` |
| `git_checkout` | 지정 브랜치·커밋 체크아웃 | `git_checkout --repository /path/to/repo --ref feature-x` | 성공 시 `0` |
| `git_branch` | 로컬 브랜치 목록 및 현재 브랜치 표시 | `git_branch --repository /path/to/repo` | 텍스트 목록(현재 브랜치 `*` 표시) |

## 5. 정상 동작 기능
- **즉시 사용 가능** – API 키·계정 없이 로컬 레포 지정만으로 모든 도구 호출 가능.  
- **다양한 diff 도구** – `unstaged`, `staged`, `ref` 용 3개의 전용 도구 제공.  
- **날짜 기반 로그 필터링** – `git_log --start_timestamp <ISO>` 형태로 기간별 커밋을 손쉽게 조회.  
- **테스트 커버리지** – 프로젝트 전체 100 % 테스트 커버리지 달성【※출처 확인 필요】.

## 6. 제한 및 미지원 기능
| 제한 항목 | 상세 내용 | 현황 |
|-----------|----------|------|
| **푸시·풀·페치** | 전혀 지원되지 않음. Issue #618(13개월째 열림) 및 PR #2961(2025 년 11월 이후 미처리) 로 확인 | 미지원 |
| **머지·리베이스** | 브랜치 생성은 가능하지만 통합 작업은 제공되지 않음 | 미지원 |

| **전송 방식** | stdio 전용; HTTP/SSE 스트리밍 미지원 | 제한 |

## 7. 보안 업데이트 및 CVE 현황

|--------|------|--------------|-----------|
| CVE‑2025‑68143 | 8.8 | `git_init` 가 임의 경로 허용 → 원격 코드 실행 가능성 | `git_init` 완전 제거 |
| CVE‑2025‑68144 | 8.1 | `git_diff`·`git_checkout` 에서 인자 인젝션 | 입력값 정화 적용 |

위 3개 CVE 모두 2025 년 12 월까지 패치 완료되었으며, 현재 배포된 버전은 해당 취약점이 제거된 상태다【※출처 확인 필요】.

## 8. 설치 및 기본 사용 방법
1. **PyPI 설치**  
   `pip install git-mcp-server`  

2. **레포지토리 지정**  
   `git-mcp-server --repository /path/to/repo`  

3. **도구 실행 예시**  

   - **상태 확인**  
     `git_status --repository /path/to/repo`  

   - **변경점 보기 (unstaged)**  
     `git_diff_unstaged --repository /path/to/repo`  

   - **커밋 만들기**  
     `git_commit --repository /path/to/repo --message "Add feature X"`  

> 모든 명령은 stdio 기반이며, 파이프라인에서 직접 호출 가능하다.

## 12. 결론
- **강점** – Zero‑dependency 경량 설계, 100 % 테스트 커버리지, 2025 년 12 월까지 모든 알려진 CVE 패치 완료, 주간 256 K 다운로드라는 높은 채택률(※출처 확인 필요).  
- **주요 한계** – 푸시·풀·머지 등 핵심 협업 기능 부재, stdio 전용 제한, 일부 기본 Git 작업(`stash·tag·blame`) 미지원.  
- **사용 권고** – **읽고 커밋** 수준의 자동화·CI 파이프라인(예: 코드 리뷰 후 자동 커밋)에는 충분히 활용 가능하지만, **전체 Git 워크플로**가 필요한 경우 `cyanheads/git-mcp-server` 혹은 GitKraken MCP Server 를 검토한다.
