# `dependencies` 라벨

`dependencies` 라벨은 **Dependabot** 이 자동으로 생성하는 Pull Request에 적용되는 라벨입니다. 이 라벨은 PR이 프로젝트의 **의존성(Dependencies)** 업데이트와 관련됨을 명시합니다.

## 주요 목적
- **자동 라벨링**: Dependabot이 생성한 PR에 자동으로 붙여, 리뷰어가 의존성 업데이트임을 즉시 파악할 수 있게 합니다.
- **필터링**: 라벨을 이용해 PR 리스트를 필터링하거나, 자동화 워크플로우(예: 병합 전략, 알림)에서 조건으로 사용할 수 있습니다.
- **정책 적용**: 조직 차원에서 의존성 업데이트에 대한 별도 정책(예: 자동 병합, 보안 검토) 적용 시 라벨을 기준으로 트리거를 설정합니다.

## 라벨이 적용되지 않을 경우
Dependabot이 라벨을 붙이지 못하는 경우는 다음과 같습니다.
1. **`dependabot.yml` 파일에 라벨 정의가 누락**
   - `.github/dependabot.yml` 에 `labels:` 섹션이 없으면 라벨이 자동으로 추가되지 않을 수 있습니다.
2. **라벨 자체가 레포지토리에 존재하지 않음**
   - GitHub 레포지토리 설정 → *Labels* 페이지에서 `dependencies` 라벨을 직접 생성해야 합니다.
3. **라벨 이름 오타**
   - `dependencies` 대신 다른 문자열을 사용하면 라벨이 적용되지 않습니다.

## 해결 방법
1. **라벨 생성**
   - 레포지토리 → *Settings* → *Labels* 로 이동 후 `New label` 버튼을 클릭합니다.
   - **Name**: `dependencies`
   - **Description** (선택): `Pull requests generated by Dependabot for dependency updates.`
   - **Color**: 원하는 색상 (예: `#0366d6`).
2. **`dependabot.yml` 확인**
   - 파일에 `labels:` 섹션이 포함되어 있는지 확인하고, 필요 시 `dependencies` 라벨을 명시합니다.
   ```yaml
   version: 2
   updates:
     - package-ecosystem: "npm"
       directory: "/"
       schedule:
         interval: "weekly"
       labels:
         - "dependencies"
   ```
3. **라벨 적용 확인**
   - Dependabot이 새 PR을 생성하면 자동으로 `dependencies` 라벨이 붙어 있는지 확인합니다.
