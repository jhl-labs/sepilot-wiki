# Dependabot Configuration and Missing Labels Fix

## Problem
When Dependabot creates a pull request, it tries to add the label `dependencies`. The repository does not have this label defined, resulting in the following error:

> The following labels could not be found: `dependencies`. Please create it before Dependabot can add it to a pull request.

## Solution Overview
1. **Create the missing label** in the repository.
2. **Update `dependabot.yml`** to reference only existing labels (or remove the label entry if you prefer not to use it).
3. Verify that Dependabot can now add the label without errors.

## Step‑by‑Step Guide
### 1. Create the `dependencies` label
1. Go to **Repository Settings → Labels**.
2. Click **New label**.
3. Fill in:
   - **Name:** `dependencies`
   - **Color:** (choose any, e.g., `#0366d6`)
   - **Description:** `Pull requests generated by Dependabot for dependency updates.`
4. Save the label.

### 2. Review and edit `dependabot.yml`
The file is typically located at the root of the repository.
```yaml
# .github/dependabot.yml
version: 2
updates:
  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"
    # Optional: add labels that already exist in the repo
    labels:
      - "dependencies"   # <-- ensure this label exists (created in step 1)
```
If you decide **not** to use the `dependencies` label, simply remove the `labels` block or replace it with other existing labels.

### 3. Commit the changes
```bash
git add .github/dependabot.yml
# If you added the label via the UI, no file change is needed for the label itself.
git commit -m "fix: ensure dependabot.yml references existing labels"
git push
```
### 4. Verify
- Open a new Dependabot PR (or trigger a manual run).
- Confirm that the PR now includes the `dependencies` label without errors.

## Additional Tips
- **Label Management:** Keep a consistent naming convention for automation labels (e.g., `dependencies`, `security`, `ci`).
- **Automation:** You can automate label creation using a repository‑initialisation script if you manage many repos.
- **Documentation:** Add a section in your repository’s contribution guide about required automation labels to avoid future issues.
