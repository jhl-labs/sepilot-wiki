## 프로젝트 개요 및 요구 사항

- **주요 기능**  
  - 실시간 오디오 파일 업로드 → 감정 라벨 반환  
  - 부정 감정 감지 시 인터벤션 로직 트리거  
  - 사용자 피드백 수집 및 재학습 파이프라인 연동

- **기술 스택**  
  - Python 3.9+  
  - `librosa`, `python_speech_features` (오디오 전처리·특징 추출)  
  - TensorFlow/Keras **또는** PyTorch (CNN 모델)  
  - Scikit‑learn (데이터 스케일링·분할)  
  - FastAPI + Uvicorn (API 서버)  
  - Docker (컨테이너화)

## 오디오 전처리 및 특징 추출
1. **파일 포맷·샘플링 레이트 통일**  
   - 일반적으로 22 050 Hz, mono 채널 사용 (`librosa.load` 기본값)  
2. **노이즈 감소·정규화**  
   - `librosa.effects.preemphasis`, `librosa.util.normalize` 등 활용 가능  
3. **특징 추출**  
   - **MFCC** (멜‑주파수 켑스트럼 계수) – 가장 널리 사용되는 음성 특징  
   - **Mel‑Spectrogram**, **Chroma**, **Zero‑Crossing Rate** 등 보조 특징을 필요에 따라 추가  
4. **특징 벡터 정규화·차원 축소**  
   - 평균·표준편차 기반 스케일링 (`sklearn.preprocessing.StandardScaler`)  
   - 차원 축소는 PCA 등 선택적으로 적용 *추가 조사가 필요합니다.*  

### 특징 추출 예시 (Python)

    import librosa
    import numpy as np

    def extract_features(file_path):
        # Load audio file (22050 Hz is standard)
        audio, sample_rate = librosa.load(file_path, res_type='kaiser_fast')
        # Extract MFCCs (40 coefficients)
        mfccs = librosa.feature.mfcc(y=audio, sr=sample_rate, n_mfcc=40)
        # Mean across time axis → fixed‑size vector
        mfccs_processed = np.mean(mfccs.T, axis=0)
        return mfccs_processed

## 모델 설계·학습
- **모델 선택 옵션**  
  - 1‑D CNN (MFCC 시계열 입력)  
  - 2‑D CNN (Mel‑Spectrogram 이미지 입력)  
  - 하이브리드 CNN‑RNN (시간적 의존성 필요 시)  
- **예시 아키텍처 (TensorFlow/Keras)**  

    from tensorflow.keras.models import Sequential
    from tensorflow.keras.layers import Dense, Dropout, Conv1D, MaxPooling1D, Flatten

    def build_model(input_shape):
        model = Sequential([
            # First Convolutional Block
            Conv1D(64, kernel_size=5, activation='relu', input_shape=input_shape),
            MaxPooling1D(pool_size=2),
            Dropout(0.3),
            # Second Convolutional Block
            Conv1D(128, kernel_size=5, activation='relu'),
            MaxPooling1D(pool_size=2),
            Dropout(0.3),
            Flatten(),
            Dense(64, activation='relu'),
            Dropout(0.3),
            Dense(3, activation='softmax')  # 3 클래스: 긍정, 중립, 부정
        ])
        model.compile(optimizer='adam',
                      loss='categorical_crossentropy',
                      metrics=['accuracy'])
        return model

## FastAPI 엔드포인트 설계
- **API 구조**  

    /health          GET    서비스 상태 확인  
    /predict         POST   오디오 파일 업로드 → 감정 라벨 반환  
    /feedback        POST   사용자 피드백 저장·재학습 트리거  

- **요청·응답 스키마 (Pydantic)**  

    from pydantic import BaseModel

    class PredictionResponse(BaseModel):
        emotion: str
        confidence: float

    class FeedbackRequest(BaseModel):
        file_id: str
        true_label: str
        comments: str | None = None

- **파일 업로드 처리**  
  - `UploadFile` 객체를 스트리밍으로 받아 임시 디스크에 저장 후 `extract_features` 호출  
- **오류 처리**  
  - 파일 포맷 검증 실패 → 415 Unsupported Media Type  
  - 내부 오류 → 500 Internal Server Error  
  - 상세 오류 메시지는 로그에 기록하고 클라이언트에는 일반 메시지만 반환
