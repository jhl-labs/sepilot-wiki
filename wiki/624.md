## 1. 문서 개요
이 문서는 **Fetch MCP Server**(이하 *fetch*)가 무엇이며, 어떻게 설치·사용할 수 있는지, 주요 기능·제한 사항 및 보안 이슈를 상세히 설명합니다.  
대상 독자는  
- AI 에이전트·플러그인 개발자  
- 자동화·노트 작성 파이프라인을 구축하려는 엔지니어  
- 보안·운영 팀  
이며, fetch를 활용해 웹 페이지를 Markdown 형태로 추출하고자 하는 모든 사용자에게 유용합니다.

## 3. 핵심 기능 및 작동 원리
- **단일 목적 도구**: URL을 입력받아 해당 페이지의 핵심 콘텐츠를 **Markdown** 형식으로 반환합니다【euno.news】.  
- **처리 파이프라인**  
  1. `httpx` 로 HTTP 요청 수행  
  2. `robots.txt` 검사로 윤리적 접근 보장  
  3. `readabilipy` 로 주요 텍스트와 메타데이터 추출  
  4. `markdownify` 로 HTML → Markdown 변환  
  5. `max_length` 파라미터에 따라 출력 길이 제한 및 청크 단위 반환【euno.news】  
- **청크 읽기**: `start_index`와 `max_length`를 조합해 대용량 페이지를 토큰 예산에 맞게 나눠 반환할 수 있습니다【euno.news】.

## 4. 설치 및 초기 설정
```bash
uvx mcp-server-fetch
```  
- 위 명령 하나로 최신 패키지를 실행 환경에 설치합니다.  
- 별도의 **API 키**가 필요 없으며, Python 3.9 이상이면 바로 사용 가능합니다.  
- 설치 전 `uv`(Universal Virtualenv) 가 설치돼 있어야 합니다(공식 문서: https://github.com/astral-sh/uv).  

## 5. 사용 방법
### 기본 호출
```bash
fetch --url https://example.com
```  
### 주요 파라미터
| 파라미터 | 필수 | 기본값 | 설명 |
|----------|------|--------|------|
| `url` | 예 | — | 가져올 대상 웹 페이지 URL【euno.news】 |
| `max_length` | 아니오 | 5000 | 반환할 Markdown 텍스트의 최대 문자 수【euno.news】 |
| `start_index` | 아니오 | 0 | 청크 단위 읽기를 위한 시작 인덱스 (대용량 페이지에 유용)【euno.news】 |
| `raw` | 아니오 | false | `true` 시 Markdown 대신 가공되지 않은 HTML 반환【euno.news】 |

### 실전 시나리오 예시
- **연구 노트 자동 요약**: `fetch --url https://arxiv.org/abs/2301.00001 --max_length 3000` 로 논문 초록을 Markdown 형태로 추출 후 LLM 요약에 활용.  
- **IDE 플러그인 연동**: VS Code 확장 프로그램이 내부적으로 `fetch`를 호출해 현재 커서 위치의 URL을 즉시 마크다운으로 변환, 문서 작성 속도 향상.

## 7. 보안 이슈
- **CVE‑2025‑65513** (SSRF, CVSS 9.3)  
  - `is_ip_private()` 함수가 사설 IP 범위를 정확히 감지하지 못해, 공격자가 `localhost`, AWS 메타데이터 엔드포인트 등 내부 리소스를 호출하도록 속일 수 있음【euno.news】.  
  - 현재까지 공식 패치가 배포되지 않았으며, 최신 2025.4.7 버전에도 동일 취약점이 존재함【euno.news】.  
- **대응 방안**  
  - 신뢰할 수 없는 외부 입력을 절대 전달하지 않음.  
  - 내부 네트워크에서만 제한된 화이트리스트 URL을 사용.  
  - 프록시·방화벽 레이어에서 추가 SSRF 방어 규칙 적용.  

## 8. 장점·단점 분석
### 장점
- **간결한 HTML‑to‑Markdown 파이프라인**으로 보일러플레이트 코드 제거【euno.news】.  
- **청크 기능**(`start_index`/`max_length`)이 토큰 예산 관리에 유리【euno.news】.  
- **robots.txt 자동 처리**로 윤리적 웹 스크래핑 지원【euno.news】.  
- **내장 프록시** 지원 및 다양한 IDE·플러그인(Claude Desktop, VS Code, Cursor 등)과의 통합 사례 다수【euno.news】.  

### 단점
- **JavaScript 렌더링 부재** → SPA·동적 페이지에서 내용이 누락될 수 있음【euno.news】.  
- **기본 `max_length` 5 000자** 제한으로 전체 기사 전체를 한 번에 가져오기 어려움【euno.news】.  
- **배치 URL 지원 미흡** – 호출당 단일 URL만 처리 가능【euno.news】.  
- **인증 헤더·쿠키 전달 불가** → 로그인·API 키가 필요한 사이트 접근 불가【euno.news】.

## 10. 평가 및 권장 사항
- **강점**: 설정이 간단하고, 신뢰할 수 있는 HTML‑to‑Markdown 변환을 제공한다는 점에서 빠른 프로토타입 제작에 적합합니다.  
- **위험**: 현재 미패치된 **CVE‑2025‑65513** SSRF 취약점이 심각하므로, 외부 입력을 직접 전달하는 공개 API·서비스에서는 사용을 **권장하지 않습니다**.  
- **권장 사용 환경**:  
  - 내부 네트워크·신뢰된 URL만 대상으로 하는 **사내 자동화 파이프라인**.  
  - 인증·프록시가 필요 없는 **공개 기사·블로그** 추출.  
- **향후 기대**: SSRF 보호 PR(#3180)이 병합되고, `max_length` 및 배치 처리 기능이 확장되면 보다 폭넓은 활용이 가능할 것으로 보입니다.

## 11. 활용 사례
1. **개인 연구·노트 자동 요약** – 논문·블로그 URL을 fetch로 Markdown 변환 후 LLM 요약에 입력.  
2. **내부 문서화 파이프라인** – 사내 위키 페이지를 정기적으로 fetch해 최신 내용으로 자동 업데이트.  
3. **IDE 플러그인 연동** – VS Code 확장에서 현재 커서 URL을 fetch 호출, 마크다운 미리보기 제공.  
4. **제한된 환경** – 프록시만 허용되는 기업 네트워크에서 간단히 웹 콘텐츠를 추출, 별도 브라우저 엔진 없이도 텍스트 확보.

## 12. 결론
Fetch MCP Server는 **간결하고 신뢰성 있는 HTML‑to‑Markdown 변환**을 제공하는 유용한 도구이며, 특히 **청크 기반 토큰 관리**와 **robots.txt 자동 처리**가 돋보입니다. 그러나 **미패치 SSRF 취약점(CVE‑2025‑65513)**이 존재하므로, **신뢰할 수 없는 외부 입력**을 다루는 서비스에서는 사용을 피하고, 내부·신뢰된 URL에 한정해 활용하는 것이 안전합니다. 향후 보안 패치와 기능 확장이 이루어지면, 보다 광범위한 자동화 시나리오에 적용 가능할 것입니다.
